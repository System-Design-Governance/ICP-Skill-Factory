# Engineering Skill Domain Map

**專案：** 領域技能治理系統 — 階段 1 產出
**版本：** v1.0 Draft
**日期：** 2026-03-13
**用途：** 作為技能分類、AI 代理能力註冊、培訓體系及工程治理的基礎分類架構

---

## 設計原則

1. **互斥性（MECE）**— 每個子領域只歸屬一個主領域，避免分類歧義
2. **可重用性** — 領域定義不綁定特定專案或客戶，適用於所有工程場景
3. **可組合性** — 領域之間可透過依賴關係組合，支援 AI 代理的技能鏈式調用
4. **可演進性** — 結構預留擴展空間，新技術領域可無縫加入
5. **雙語命名** — 每個領域同時具備英文 ID 和中文名稱，供系統與人類共用

### D01 — OT 資訊安全（OT Cybersecurity）

OT/ICS 環境的安全架構、風險評估、合規治理與日常防護。遵循 IEC 62443 生命週期。

| 子領域 ID | 子領域名稱 | 說明 | 技能範例 |
|-----------|-----------|------|---------|
| D01.1 | 安全架構設計 | Zone/Conduit 架構、縱深防禦、安全區域劃分 | Zone/Conduit 拓撲設計、防火牆規則規劃、DMZ 設計 |
| D01.2 | 風險評估與威脅建模 | 資產辨識、威脅分析、風險量化與處置 | STRIDE 分析、DREAD 評分、資產清冊建立、風險矩陣 |
| D01.3 | 合規與稽核 | IEC 62443 / ISO 27001 合規評估、差距分析 | SL-T 評估、控制項對照、合規差距矩陣、稽核準備 |
| D01.4 | 安全監控與事件回應 | 即時監控、異常偵測、事件處置流程 | SIEM 配置、告警規則設計、IR 程序撰寫、鑑識分析 |
| D01.5 | 安全加固與組態管理 | 設備安全加固、存取控制、補丁管理 | 設備 Hardening、帳號權限管理、安全基線配置 |
| D01.6 | 供應鏈安全 | 供應商風險評估、元件安全驗證 | 供應商安全問卷、SBOM 分析、第三方風險評分 |

### D02 — 系統架構設計（System Architecture）

OT/IT 系統的整體架構規劃，包含網路拓撲、系統介面、通訊架構與部署策略。

| 子領域 ID | 子領域名稱 | 說明 | 技能範例 |
|-----------|-----------|------|---------|
| D02.1 | OT 網路架構 | 工業網路拓撲設計、VLAN 規劃、冗餘架構 | 環形網路設計、RSTP 配置、網路分段規劃 |
| D02.2 | 系統介面設計 | 子系統間的介面定義與資料交換規範 | 介面規範書撰寫（ICD）、API 定義、資料流圖 |
| D02.3 | 通訊架構 | 工業通訊協定選用與架構設計 | Modbus/IEC 61850/DNP3/OPC UA 架構規劃 |
| D02.4 | 高可用與冗餘設計 | 系統可靠性設計、容錯機制、災難復原 | 雙機熱備設計、故障切換策略、RTO/RPO 規劃 |
| D02.5 | 雲端與邊緣架構 | 雲地混合架構、邊緣計算部署 | 邊緣閘道規劃、雲端資料同步設計、混合架構選型 |
| D02.6 | 架構評審與決策 | 架構決策記錄（ADR）、技術選型評估 | ADR 撰寫、技術選型矩陣、架構審查會議主持 |

### D03 — 電力系統工程（Power System Engineering）

電力系統的分析、設計、模擬，涵蓋傳統電網與再生能源系統。

| 子領域 ID | 子領域名稱 | 說明 | 技能範例 |
|-----------|-----------|------|---------|

| D03.2 | 再生能源整合 | 太陽能/風力併網設計、功率預測與調度 | PV 系統設計、風場併網分析、功率預測模型 |
| D03.3 | 儲能系統工程 | 電池儲能系統設計、控制策略、經濟評估 | BESS 容量規劃、充放電策略設計、劣化模型分析 |

| D03.6 | 電力系統模擬 | 系統建模、暫態模擬、事件重現 | ETAP/PSS/E 建模、暫態穩定度模擬、事故重現分析 |

## 設計決策記錄

### ADR-001：將保護工程獨立為主領域

**決策：** 保護工程（D04）從電力系統工程（D03）獨立出來。
**理由：** 保護工程有獨立的工具鏈（繼電器測試儀、保護協調軟體）、獨立的專業認證、且在組織中通常由專職工程師負責。獨立後更利於技能評估與培訓路徑設計。

### ADR-002：將能源資料平台獨立為主領域

**決策：** 能源資料平台（D12）從控制系統工程（D05）獨立出來。
**理由：** 資料平台工程越來越成為獨立的專業領域，涉及時序資料庫、ML 模型、資料治理等，與傳統 SCADA 控制工程技能集差異顯著。隨著 VPP 和 DERMS 的發展，此領域將持續擴大。

### ADR-003：新增工程自動化主領域

**決策：** 新增 D13 工程自動化，不包含在原始必要覆蓋清單中。
**理由：** 組織明確將 AI 代理與自動化工作流程列為技能系統的使用者。工程自動化本身也是一項需要治理的能力領域，且預計將快速成長。
