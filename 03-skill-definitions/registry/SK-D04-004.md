## Description

Develop protection logic diagrams (Boolean logic expressions, trip matrices, interlocking sequences) for relay-based and IED-based protection schemes. This skill encompasses translating protection coordination requirements and relay settings into executable logic sequences that define trip signals, blocking conditions, and permissive interlocking. Covers both hardwired relay logic and programmable IED logic. Results feed into IEC 61850 SCL configuration (SK-D05-010) for digital substation implementations and inform SCADA logic implementation (SK-D05-003).

---

## Inputs

- Protection coordination scheme design (from SK-D04-001, SK-D04-002)
- Relay settings and time delays (overcurrent, distance, differential thresholds)
- Equipment protection strategy (transformer protection scheme, line protection, bus protection)
- Digital relay/IED specifications and logic programming capabilities
- Control system interlocking requirements (breaker lockout, synchro-check, reclosure logic)
- Standard control sequences (ANSI/IEEE device numbering, IEC 61850 logic node definitions)
- Cybersecurity constraints (voting logic for critical protection functions)

---

## Outputs

- Protection logic diagram (graphical representation: AND/OR/NOT logic gates or hardwired diagram)
- Trip matrix table (input conditions → trip signal output mapping)
- Boolean logic expressions (formal notation for each trip function)
- Interlocking sequence documentation (breaker trip hierarchy, blocking conditions)
- IED configuration parameter list (for programmable devices)
- Logic timing diagram (trip signal propagation and time delays)
- Validation checklist (logic coverage, race conditions, timing hazards)
- IEC 61850 Logical Node mapping (for SCL configuration input)

## IEC 62443 Lifecycle Stages

- **Planning & Requirements**: Define protection logic requirements and security objectives
- **Design & Implementation**: Develop logic diagrams and program IED logic blocks
- **Verification & Validation**: Test logic sequences through simulation and hardware-in-loop testing
- **Operations & Maintenance**: Monitor logic operation; update for configuration changes
- **Decommissioning**: Archive logic diagrams and retire obsolete relay logic

## Roles

- **Design Engineer (DES)**: Primary owner; develops logic diagrams and specifications
- **Protection Engineer**: Reviews logic for coordination and completeness
- **IED Configuration Engineer**: Translates logic diagrams into IED parameter settings
- **Controls Engineer**: Integrates logic with SCADA/EMS control systems
- **Commissioning Engineer**: Tests and validates logic during factory and site acceptance testing

## Acceptance Criteria

1. Logic diagrams clearly represent all primary and backup protection functions
2. Trip matrices completely map all input conditions (threshold crossings, time delays) to trip outputs
3. Boolean logic expressions are formally correct and unambiguous (no logical hazards or race conditions)
4. Interlocking sequences prevent unwanted simultaneous trips and ensure hierarchical fault isolation
5. Timing diagram demonstrates adequate delay coordination (0.2–0.5 sec spacing between zone trips)
6. All ANSI/IEEE device numbers assigned correctly and cross-referenced in one-line diagram
7. IEC 61850 Logical Node mapping provided for IED-based implementations
8. Logic validation confirms no orphaned signals, floating inputs, or unreachable logic states
9. Contingency analysis confirms logic integrity if any single input/sensor fails
10. Sign-off by both Design Engineer and Protection Engineer
