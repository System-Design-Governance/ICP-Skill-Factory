## Description

Prepare responses to Requests for Information (RFI) from prospective customers/employers, providing structured information about ICP's OT/ICS cybersecurity capabilities, relevant project experience, technical qualifications, and service offerings. RFI responses are typically earlier and lighter-weight than full tender proposals — they aim to establish ICP's qualification for subsequent tender invitation rather than proposing a specific technical solution. This skill covers RFI analysis, response authoring, and submission coordination; the full technical proposal that may follow a successful RFI falls under SK-D14-008, and the detailed security requirements definition for tender falls under SK-D14-010.

## Inputs

- Customer/employer RFI document: questions, evaluation criteria, submission format requirements
- ICP capability statements: certifications (IEC 62443, ISO 27001), company profile, service portfolio
- Relevant project references: past project descriptions, client testimonials, case studies
- Team qualifications: personnel certifications, experience summaries, key personnel CVs
- ICP standard RFI response templates (if available)

## Outputs

- RFI Response Document:
  - Company overview: ICP profile, organizational structure, cybersecurity service portfolio
  - Capability statements: per-RFI-question responses demonstrating relevant capabilities
  - Relevant experience: project references with scope descriptions, client names (where permitted), outcomes achieved
  - Technical qualifications: team certifications, technology partnerships, vendor authorizations
  - IEC 62443 alignment: demonstration of service delivery aligned with IEC 62443 lifecycle framework
  - Compliance matrix: point-by-point response to each RFI question with section reference
- RFI Submission Package: formatted per customer requirements (document format, page limits, appendices)
- RFI Lessons Learned: post-submission notes on customer focus areas for subsequent tender preparation

## Roles

- Pre-Gate Design Support Role (GOV-SDP) — primary: authors RFI response technical content
- PM (Project Manager) — manages submission timeline, coordinates with business development
- SAC (Security Architect) — contributes security capability content and IEC 62443 alignment narrative
- Head of System Design (GOV-SDP) — reviews RFI response for brand alignment and technical accuracy

## Dependencies

### Hard Dependencies (Required Before Execution)

- None — RFI response is typically the earliest customer engagement activity, requiring only ICP organizational information

### Soft Dependencies (Enhances Effectiveness)

- SK-D14-001: Requirements Specification Development — if RFI includes scope-specific questions, requirements understanding helps
- ICP project reference database — relevant experience lookup

## Automation Potential

**Level:** Partial

Capability statement generation from organizational databases can be automated. Project reference lookup and formatting can be automated. Compliance matrix template generation can be automated. However, RFI question interpretation requires contextual understanding, response tailoring to specific customer context requires communication judgment, and strategic positioning (how to differentiate from competitors) requires human expertise.

## Acceptance Criteria

1. RFI response addresses every question/section in the customer RFI — compliance matrix shows 100% coverage
2. Capability statements are specific and evidence-based: certifications cited, project numbers referenced, team qualifications listed
3. Relevant experience includes minimum 3 project references with scope descriptions aligned to RFI focus areas
4. IEC 62443 alignment demonstrated: service delivery methodology linked to IEC 62443 lifecycle framework
5. Submission format complies with customer requirements: page limits, document format, naming conventions
6. RFI response reviewed and approved by Head of System Design before submission

## Estimated Effort

| Level | Effort | Notes |
|-------|--------|-------|
| Junior (< 2 yr) | 3–5 person-days | Assumes standard RFI with 10–20 questions; includes analysis, response authoring, formatting, and review |
| Senior (5+ yr) | 2–3 person-days | Same scope; senior leverages existing capability statements and reference library |

Notes: Effort varies with RFI complexity and question count. Simple capability RFIs may take 1–2 days; detailed technical RFIs with specific scenario questions may require 5+ days. Reuse of prior RFI responses significantly reduces effort. International RFIs requiring translation add overhead. RFI lessons learned should be captured to improve subsequent responses.
