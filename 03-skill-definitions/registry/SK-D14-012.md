## Description

Identify, document, and catalog the existing infrastructure at the project site, covering automation systems, network equipment, servers, security appliances, and communication infrastructure already in place. This Pre-Gate 0 inventory establishes the brownfield baseline that informs feasibility assessment, integration requirements, and scope definition. The boundary of this skill is existing equipment discovery and documentation during the pre-project phase; the formal asset inventory with criticality ratings and zone assignments created during the project falls under SK-D01-005, and the physical site constraints fall under SK-D14-011. Existing infrastructure inventory is a coarser, faster assessment focused on "what exists" rather than the detailed "what matters for security" analysis of SK-D01-005.

## Inputs

- Customer/employer site documentation: as-built drawings, equipment lists, network diagrams, maintenance records
- Site survey access (from SK-D14-011 — often conducted in the same site visit)
- Customer/employer technical contacts for equipment identification
- Vendor documentation for existing systems (model numbers, firmware versions where accessible)

## Outputs

- Existing Infrastructure Inventory Register:
  - Automation systems: PLCs/RTUs, HMIs, SCADA servers, historians, engineering workstations — manufacturer, model, firmware version, age
  - Network equipment: switches, routers, firewalls, media converters — manufacturer, model, configuration status (managed/unmanaged)
  - Server infrastructure: physical/virtual servers, operating systems, virtualization platforms
  - Communication infrastructure: cable plant type (copper/fiber), cable routes, patch panels, wireless APs
  - Security appliances: existing firewalls, IDS/IPS, antivirus servers, SIEM (if any)
  - Physical infrastructure: cabinets, racks, UPS, environmental controls
  - Per item: location, condition assessment (Operational / Degraded / End-of-Life), connectivity status
- Integration Impact Assessment: preliminary analysis of how existing infrastructure affects new system design (compatible/incompatible equipment, upgrade candidates, replacement requirements)
- End-of-Life Risk Register: equipment approaching or past end-of-life/end-of-support dates with replacement urgency assessment
- Gap Summary: infrastructure categories where existing equipment is absent or insufficient for the proposed solution

## Tools

- Asset discovery tools (SNMP walk, Nmap — for network-connected equipment)
- Serial/console access tools (for legacy equipment identification)
- Camera/documentation tools (equipment labels, cabinet photographs)
- Vendor product identification databases (model/firmware lookup)
- Spreadsheet tools (Excel — inventory register)

## IEC 62443 Lifecycle Stages

## Roles

- Pre-Gate Design Support Role (GOV-SDP) — primary: leads inventory effort, documents findings
- DES (Design Engineer) — supports equipment identification, captures technical details
- SYS (System Architect) — assesses integration implications of existing infrastructure
- Customer/Employer Technical Contact — provides site knowledge, equipment access, and historical documentation

## Dependencies

### Hard Dependencies (Required Before Execution)

- None — infrastructure inventory can be conducted as soon as site access is arranged

### Soft Dependencies (Enhances Effectiveness)

- SK-D14-011: Site Survey and Constraint Documentation — often conducted in the same site visit; physical constraints contextualize equipment placement
- SK-D14-001: Requirements Specification Development — project scope focuses inventory on relevant infrastructure categories

## Automation Potential

**Level:** Partial

Network-connected equipment discovery can be partially automated (SNMP walk, Nmap scanning, ARP table extraction). Equipment model/firmware identification can be automated for devices with standard management interfaces. However, legacy equipment without network management requires physical inspection, cabinet-level inventory requires on-site human presence, condition assessment requires expert judgment, and integration impact analysis requires design knowledge.

## Acceptance Criteria

1. Inventory register covers all six infrastructure categories: automation, network, server, communication, security, and physical — no category empty without documented rationale
2. Every identified item has: location, manufacturer, model, condition assessment, and connectivity status — no items with incomplete records
3. End-of-life risk register produced: all equipment past or within 2 years of end-of-support identified with replacement urgency classification
4. Integration impact assessment completed: existing infrastructure compatibility with proposed solution evaluated per major system category
5. Gap summary identifies infrastructure categories requiring new deployment — sufficient for CBOM scoping (SK-D14-005 ⏳)
6. Inventory register format compatible with SK-D01-005 (Asset Inventory Development) — existing infrastructure items can be imported into the formal project asset inventory during R0/R1

## Estimated Effort

| Level | Effort | Notes |
|-------|--------|-------|
| Junior (< 2 yr) | 3–5 person-days | Assumes single-site, ~50–100 existing devices; includes on-site discovery (1–2 days), documentation, and impact assessment |
| Senior (5+ yr) | 2–3 person-days | Same scope; senior conducts efficient focused assessment with rapid model identification |

Notes: Effort scales with site size and equipment count. Large industrial sites with hundreds of legacy devices may require 5–10 person-days. Greenfield projects require minimal effort (document only utility connections and building infrastructure). Often combined with SK-D14-011 site survey in a single site visit. Poor existing documentation significantly increases discovery effort.
