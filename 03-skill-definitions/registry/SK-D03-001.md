## Description

Power Flow Analysis is the systematic execution of electrical power distribution analysis on network topologies to characterize steady-state voltage profiles, real and reactive power flows, and equipment loading conditions. The skill performs load flow calculations under normal operating conditions and contingency scenarios using industry-standard simulation platforms (ETAP, PSS/E, or equivalent) to establish the electrical baseline for security zone design and equipment specification.

## Description — Scope

This skill encompasses the execution phase of power flow studies, including network model configuration, load and generation data input, solver execution, convergence troubleshooting, and result interpretation. The skill boundary does not include power system modeling tool installation, configuration management systems, or historical data archival—these are addressed in support SCs. Results directly inform zone boundary placement decisions in SK-D01-001 and equipment sizing parameters for electrical panel design activities.

## Acceptance Criteria

- Power flow model convergence achieved for base case and minimum five defined contingency scenarios
- Voltage magnitude maintained within ±5% of nominal at all network buses in normal operation
- Maximum equipment loading percentage identified and documented for all critical network elements
- Reactive power margins confirmed adequate for voltage stability across network
- Contingency scenario results provided in tabular and graphical formats suitable for engineering review
- All numerical results traceable to solver output logs with timestamp documentation

## Standards

Power flow analysis is foundational to IEC 62443 security control implementation. Accurate characterization of normal operating electrical parameters enables correct specification of protective device settings (SL-C controls), detection thresholds for anomaly-based intrusion detection, and baseline establishment for operational technology monitoring systems. The electrical boundary determination outputs enable proper security zone segregation and network segmentation architectures.

## Description — Knowledge Requirements

- AC circuit theory and three-phase power system fundamentals
- Power flow equations (Newton-Raphson, Gauss-Seidel solver methods)
- Network topology representation and modeling conventions
- IEEE and IEC power system standards (IEEE 1415, IEC 60038)
- Contingency scenario classification and N-1 criteria
- Hands-on proficiency with ETAP or PSS/E user interface and result interpretation
- Voltage stability and reactive power compensation concepts

## Roles

- Proficiency in power flow simulation software operation and troubleshooting
- Ability to interpret voltage profiles, power flows, and equipment loading reports
- Competency in contingency scenario design and selection criteria
- Technical communication of electrical results to non-specialist stakeholders
- Problem-solving for convergence failures and unusual loading patterns
- Data validation and error checking in network model inputs

## Tools

- ETAP Power Systems or Siemens PSS/E as primary simulation platforms
- Network diagram tools for topology visualization
- Spreadsheet analysis for result post-processing and tabulation
- Contingency analysis modules within simulation platforms
- Load flow optimization functions for scenario evaluation
- Export/report generation functions for stakeholder communication

## Description — Process Steps

1. Establish network topology and connectivity within simulation platform
2. Input transmission line, transformer, and distribution feeder parameters
3. Configure load and generation profiles for peak, normal, and off-peak conditions
4. Execute base case power flow solver and verify convergence
5. Define contingency scenarios (single element outages) and execute contingency analysis
6. Review and validate all nodal voltages, power flows, and branch loading percentages
7. Troubleshoot non-convergent cases through load adjustment or network reconfiguration
8. Generate comprehensive reports with voltage profiles, power flow diagrams, and loading tables
9. Perform sensitivity analysis on critical load or generation variations
10. Document results with source data traceability and analysis date/time stamps

## Description — Decision Logic

Power flow analysis results guide several key security architecture decisions:
- Zone boundary placement: zones defined at network points with lowest power exchange rates
- Equipment ratings: transformer and breaker ratings selected based on maximum contingency loading
- Monitoring thresholds: abnormal load detection algorithms calibrated to maximum normal operating flows
- Reserve requirements: spinning reserve and voltage control capacity sized from contingency scenarios
- Protection settings: coordination of protective devices based on short circuit current from power flow results

## Dependencies

- SK-D01-001 (Security Zone Definition) - receives zone boundary inputs from power flow analysis
- SK-D03-002 (Short Circuit Current Analysis) - uses network model and operating conditions from power flow analysis
- SK-D04-001 (Overcurrent Protection Coordination) - requires maximum loading scenarios from power flow results
- SK-D06-005 (Wire Sizing Calculation) - needs conductor ampacity requirements from contingency loading

## Description — Common Errors

- Assuming base case convergence implies contingency stability—contingencies must be explicitly analyzed
- Over-relying on single load profile without considering seasonal and diurnal variations
- Neglecting voltage stability under light-load conditions
- Treating simulation results as deterministic without sensitivity analysis
- Failing to validate network model against field measurements before analysis
- Confusing apparent power, real power, and reactive power requirements in load specification

## Acceptance Criteria — Quality Metrics

- Model convergence achieved in ≥95% of defined scenarios
- Voltage margin (difference between calculated and limit voltage) documented for all contingencies
- Report completeness verified: all network buses and branches included in results
- Stakeholder sign-off obtained on zone boundary recommendations from power flow analysis
- Traceability matrix linking analysis assumptions to final engineering recommendations

## Outputs

- Power flow analysis report documenting base case and contingency voltage profiles
- Equipment loading summary table showing maximum sustained amperage for each critical element
- Network one-line diagram annotated with normal and contingency power flows
- Zone boundary recommendations based on minimum inter-zone power exchange rates
- Sensitivity analysis results showing system behavior under load ±10% variation
- Solver convergence logs and timestamp documentation of analysis completion
- Engineering sign-off on electrical baseline for subsequent security zone design activities
