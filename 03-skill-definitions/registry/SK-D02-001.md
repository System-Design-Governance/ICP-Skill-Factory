## Description

Design the physical and logical OT network topology including switch placement, VLAN segmentation, redundancy paths, and Purdue model layer mapping to establish a secure, resilient, and operationally sound network infrastructure foundation.

## Description — Business Value

Effective network topology design ensures reliable industrial process communication, enables security segmentation at the network layer, supports system scalability, and reduces implementation risks through validated architectural patterns. Organizations with well-designed OT topologies experience improved system uptime, faster incident response, and reduced unplanned downtime costs.

- Logical network segmentation (VLAN structure, subnet design)

## Description — Detailed Definition

The practitioner shall:

1. **Map system functions to Purdue layers:** Identify which OT functions (field devices, controllers, edge servers, MES, ERP) reside at each Purdue level, documenting the communication flows between layers.

2. **Design VLAN segmentation:** Define VLAN boundaries aligned to functional zones, engineering zones, and security domains, ensuring isolation of critical control loops from general-purpose networks.

3. **Specify switch placement:** Determine physical location, redundancy, and connectivity patterns for network switches, including aggregation, access, and edge layers, to minimize single points of failure and latency.

4. **Design redundancy architecture:** Select and document redundancy strategy (ring topology with rapid failover, dual paths with active-active load sharing, mesh interconnect for critical links) with recovery time objectives.

5. **Define IP addressing scheme:** Design hierarchical IP addressing aligned to VLANs, with documented addressing plans for each functional zone and spare capacity for future expansion.

6. **Specify uplink and trunk architecture:** Define inter-switch connections (trunk links, redundant uplinks) with bandwidth allocation, failover mechanisms, and performance guarantees.

7. **Document network resilience:** Specify recovery time objective (RTO), recovery point objective (RPO), and validate topology against these objectives through network simulation or calculation.

## Acceptance Criteria

- Network topology diagram shows all network segments, device interconnections, and redundancy paths with clear labeling.
- VLAN allocation table specifies VLAN ID, name, purpose, subnet range, and security level for each segment.
- Purdue layer mapping document identifies all OT functions and their assigned network layer with justification.
- Redundancy design specification states failover mechanism, recovery time, and validation evidence (simulation log or test result).
- IP addressing plan allocates addresses to each segment with documented spare capacity and no overlapping ranges.
- Uplink bandwidth calculations verify that aggregation and trunk links support peak industrial communication loads without congestion.
- Design rationale document explains topology choices relative to system requirements, site constraints, and vendor equipment capabilities.

## Description — Safety Considerations

- **Network Segmentation:** Topology must support isolation of critical control networks from information technology networks to limit attack surface.
- **Redundancy for Availability:** Dual paths and failover mechanisms protect against single-link failures that could impact process safety.
- **Electromagnetic Compatibility:** Cabling and shielding strategies must comply with industrial EMC standards (IEC 61000-6-2, IEC 61000-6-4).
- **Surge Protection:** Network topology design coordinates with surge protection strategy at network ingress points.

## Standards

- **IEC 62443-3-3 Zone Definition (A.7):** Network topology must support security zones and conduits as defined in protection strategy.
- **IEC 62443-1-1 Governance:** Architecture decisions are subject to change control and design review as per governance process.
- **Industry Standards:** Design should align with ISA/IEC 62443, NIST Cybersecurity Framework, and applicable sector-specific standards (e.g., NERC for power, API for oil & gas).

## Description — Common Pitfalls

- **Oversimplified Topology:** Flat networks without segmentation increase attack surface; ring topologies may create single-point failures if not properly designed.
- **Inadequate Redundancy Planning:** Backup paths that mirror primary paths create symmetric vulnerabilities; redundancy strategy must account for correlated failure modes.
- **Bandwidth Underestimation:** OT networks supporting high-frequency data (e.g., motion control) require precise bandwidth calculations; underestimated bandwidth leads to packet loss and process instability.
- **Ignoring Legacy Equipment:** Network design must accommodate legacy devices with limited protocol support; retrofitting topology for legacy constraints is costly and error-prone.
- **Insufficient Documentation:** Topology changes during implementation without documentation creates knowledge gaps; maintain design-to-as-built traceability.

## Description — Practical Application

**Example: Manufacturing Plant OT Network Topology**

A discrete manufacturing facility with three production areas (Press Shop, Assembly, Packaging) and a centralized MES:

- **Purdue Level 0 (Field):** Press controllers, Assembly robots, Packaging sensors → Purdue Level 1 (Edge)
- **Purdue Level 1:** Local control cabinets with local PLCs/RTUs, each with dedicated VLAN (VLAN 101, 102, 103)
- **Purdue Level 2:** Plant MES and HMI servers on VLAN 200, aggregation switch at plant level
- **Purdue Level 3:** Plant supervisory server on VLAN 300
- **Purdue Level 4/5:** Corporate ERP on separate IT network with air gap or controlled gateway

**Topology Design:**
- Ring topology at Level 1 (3 edge switches), each with dual uplinks to Level 2 aggregation switch
- Level 2 aggregation switch has dual uplinks to Level 3 supervisory server
- Redundancy: Primary uplink on Port 1, backup on Port 2; failover time < 50 ms via rapid spanning tree

**VLAN Structure:**
| VLAN | Name | Subnet | Purpose |
|------|------|--------|---------|
| 101 | Press-Shop | 192.168.101.0/24 | Press control loop |
| 102 | Assembly | 192.168.102.0/24 | Robot and assembly sensors |
| 103 | Packaging | 192.168.103.0/24 | Packaging line devices |
| 200 | MES | 192.168.200.0/24 | Production planning and data |
| 300 | Supervisory | 192.168.300.0/24 | Plant monitoring |
