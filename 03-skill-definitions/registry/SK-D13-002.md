## Description

The Document Generator Development skill creates automated systems that produce engineering deliverables (point lists, wiring schedules, test reports, compliance matrices) from structured data sources, eliminating manual compilation and formatting tasks. This capability integrates template design, data source connectivity, multi-format output handling (DOCX, XLSX, PDF), and quality validation to ensure generated documents meet engineering and compliance standards. By automating the production of engineering documentation, this skill accelerates delivery of project artifacts, reduces transcription errors, and enables rapid updates when underlying design data changes. The generator supports multiple document types across project disciplines, providing a unified framework for consistent deliverable production throughout commissioning, design, and maintenance phases.

## Inputs

- Structured engineering data sources (CAD extracts, database queries, configuration management systems)
- Document template specifications (layout, formatting, required sections)
- Mapping definitions (data-to-template field associations)
- Quality validation rules and acceptance criteria for generated documents
- Output format specifications (DOCX, XLSX, PDF rendering requirements)
- Organizational style guides and document standards

## Outputs

- Document Generator Implementation (software module or application)
- Template Library (DOCX, XLSX, PDF templates for all supported document types)
- Data Source Integration Specifications and connectors
- Quality Validation Framework (automated document review rules)
- Generator User Guide and Administration Documentation
- Example Generated Documents (one per template type)
- Template Maintenance Procedures and versioning guidelines

## IEC 62443 Lifecycle Stages

- **R0 (Initiation)**: Deliverable types identified; template requirements documented
- **R1 (Concept)**: Generator architecture and data flow designed; templates prototyped
- **R2 (Definition)**: Templates finalized; data source connectors specified; validation rules defined
- **R3 (Implementation)**: Generator deployed; templates tested; users trained
- **R4 (Verification)**: Generated documents reviewed for accuracy; quality metrics tracked

## Roles

- **QA** (Quality Assurance): Document generation testing, output validation, compliance verification
- **DOC** (Documentation Specialist): Template content development, user guide authorship

## Acceptance Criteria

1. Document generators automatically produce all required deliverable types (point lists, wiring schedules, test reports, compliance matrices) from structured input data
2. Generated documents are consistently formatted and properly structured according to templates without manual post-processing
3. Templates support all required output formats (DOCX, XLSX, PDF) with correct rendering and pagination
4. Data integration from multiple sources (CAD, databases, configuration systems) is functional and verified
5. Quality validation framework automatically detects missing fields, formatting errors, and data inconsistencies; non-conforming documents are flagged for manual review
6. Automated generator reduces manual document creation effort by at least 70% while maintaining or improving accuracy
