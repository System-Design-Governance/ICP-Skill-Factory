## Description

Configure Distributed Energy Resource Management System (DERMS) strategies for DER dispatch, curtailment control, and optimization within Virtual Power Plant (VPP) and aggregation frameworks. DERMS configuration implements dispatch algorithms (from SK-D03-006: VPP Dispatch Algorithm Design) and aggregation strategies (from SK-D03-007: DER Aggregation Strategy Design) through operational settings, setpoint logic, and constraint matrices. Configuration encompasses DER registration and operational envelope constraints (active power, reactive power, frequency support), grid service definitions and priority rules, unit commitment logic, and real-time dispatch optimization. Security considerations include DER communication security (per SK-D01-016), setpoint integrity verification (per SK-D10-002), and change management (per SK-D10-003).

## Inputs

- VPP Dispatch Algorithm Design (from SK-D03-006 ⏳: VPP Dispatch Algorithm Design)
- DER Aggregation Strategy Design (from SK-D03-007 ⏳: DER Aggregation Strategy Design)
- DERMS Platform Specification: vendor product (e.g., DNV GL Risco, AutoGrid, Stem), available optimization modules
- DER Fleet Inventory: device type, location, capacity (kW/kVAr), response time, communication protocol per resource
- Operational Envelope Data: power limits, ramping rates, minimum generation, reactive power capability, frequency support parameters
- Grid Service Requirements: frequency regulation, voltage support, demand response, congestion relief, reserve capacity
- Communication Network Architecture (from SK-D02-001: OT Network Topology Design) and bandwidth constraints
- Market and Regulatory Requirements: compensation structure, service pricing, reliability standards (e.g., NERC, utility tariffs)

## Outputs

- DERMS Configuration Parameter Table: DER registration list, operational envelope constraints, priority rules for dispatch
- Dispatch Algorithm Implementation Specification: setpoint calculation logic, unit commitment constraints, economic dispatch constraints
- Grid Service Definition Matrix: service type, DER eligibility criteria, compensation mechanism, activation logic
- Aggregation Group Configuration: resource clustering strategy, communication topology, backup DER substitution rules
- DER Communication Security Configuration (per SK-D01-016): authentication/encryption per DER type, timeout/watchdog settings
- Performance Monitoring and Audit Strategy: dispatch efficiency metrics, grid service delivery verification, change audit trail

## Tools

- DERMS Platform Configuration Tools: vendor-supplied optimizer and parameter setting interfaces
- ETAP / PSS/E (dispatch validation, optimal power flow simulation, stability analysis with DER participation)
- Python / MATLAB (dispatch algorithm development, optimization solver integration, sensitivity analysis)
- Spreadsheet / structured markdown (DER inventory, operational envelope matrices, service definitions)
- Network simulation tools (latency validation for real-time dispatch commands)

- SYS (System Architect) — primary: designs aggregation strategy, defines dispatch algorithm, assigns grid service participation
- DES (Design Engineer) — configures DERMS platform, develops signal mapping, implements communication protocol bindings
- TST (Test Engineer) — validates dispatch performance through simulation and pilot field testing
- OPS (Operations Engineer) — provides DER operational constraints, monitors dispatch performance, manages market/regulatory compliance

### Soft Dependencies (Enhances Effectiveness)

- SK-D02-001 ⏳: OT Network Topology Design — network constraints affecting real-time dispatch communication

## Acceptance Criteria

1. All DERs in the fleet are registered in DERMS with correct capacity, location, and communication protocol
2. Operational envelope constraints (power limits, ramp rates, frequency support parameters) are set per manufacturer specifications and validated against field data
3. Dispatch algorithm implementation in DERMS matches the algorithm design (SK-D03-006) with documented verification
4. Grid service priority rules are clearly documented and reflect regulatory/market requirements
5. Aggregation group communication topology is tested end-to-end with latency and reliability validation
6. DER-DERMS communication security (authentication, encryption, timeout watchdog) is configured per SK-D01-016
7. Dispatch setpoint changes are protected by change control and audit logging per SK-D10-003 and SK-D01-024
8. Configuration has been reviewed and approved by both SYS and OPS, and validated through pilot field testing
