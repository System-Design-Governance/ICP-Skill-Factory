## Description

Execute the Site Acceptance Test (SAT) at the project site after system installation, verifying that all security and functional requirements operate correctly in the production environment — including site-specific configurations, physical security controls, network connectivity, and integration with existing employer infrastructure. SAT validates that the system as-installed meets the same criteria verified during FAT, plus site-specific conditions that could not be tested in the factory. Under ICP governance (GOV-SD), SAT completion is a Gate 3 prerequisite for system handover. Per BOUNDARY-007, SAT is site testing post-installation, distinct from FAT (factory, pre-shipment) and SIT (end-to-end integration with employer existing systems requiring employer participation). This skill covers SAT execution and result documentation; the SAT procedure development falls under SK-D08-003 ⏳, and the security-specific acceptance testing (SR verification) falls under SK-D08-005 ⏳.

## Inputs

- SAT Procedure Document (from SK-D08-003 ⏳) — defines test cases, pass/fail criteria, and execution sequence for site testing
- FAT Results Report — baseline results from factory testing; SAT verifies site-environment consistency
- As-built system documentation: actual device configurations, IP addresses, cable schedules, installed firmware versions
- Site-specific conditions: physical environment, existing infrastructure, employer network policies, electromagnetic environment
- Security design documentation: Zone/Conduit architecture (SK-D01-001), hardening configurations (SK-D01-019), access control policy (SK-D01-020)
- Customer/employer SAT witness requirements and acceptance criteria
- IEC 62443-2-4 site verification requirements

## Outputs

- SAT Execution Report (per ID14 exemplar format):
  - Test execution summary: total tests, pass/fail/blocked/deferred counts
  - Per test case: test ID, execution date, tester, actual result, pass/fail status, evidence (screenshots, log excerpts, scan results)
  - Site-specific verification results:
    - Physical security: cabinet locks, physical port status, environmental controls
    - Network connectivity: actual routes, latency measurements, firewall rule verification in production
    - Integration verification: communication with employer systems, SCADA data flow confirmation
    - Performance verification: system response times under actual load conditions
  - Deviation log: differences between FAT and SAT results with root cause analysis
  - Defect register: all failures with severity classification, assigned owner, and remediation status
- SAT Completion Certificate: formal sign-off document confirming all mandatory test cases passed or deferred with approved compensating controls
- Punch List: outstanding items requiring remediation before final handover, with priority and deadline per item
- Updated As-Built Documentation: corrections to design documentation discovered during SAT (actual vs. planned configurations)

## Automation Potential

**Level:** Partial

Network connectivity verification and configuration auditing can be partially automated (scripted Nmap scans, configuration comparison tools). Test result recording and pass/fail tracking can be automated through test management platforms. However, physical security inspection requires on-site human verification, integration testing with employer systems requires interactive troubleshooting, performance testing under actual conditions requires contextual interpretation, and deviation analysis requires engineering judgment to determine root cause and appropriate remediation.

## Acceptance Criteria

1. All mandatory SAT test cases executed with results recorded: 100% coverage of security-critical test categories (no skipped mandatory tests without formal deviation approval)
2. Pass rate meets Gate 3 threshold: all Critical and High severity test cases pass; Medium severity failures have documented compensating controls with Security Engineering Role approval
3. Site-specific conditions verified: physical security, network connectivity in production, and integration with employer systems confirmed operational
4. FAT-to-SAT deviation analysis completed: all differences between factory and site results documented with root cause and resolution
5. SAT Completion Certificate signed by Security Engineering Role and customer/employer representative
6. Punch list produced with priority assignments: all outstanding items have owner, deadline, and no Critical items remain open at Gate 3 submission

## Estimated Effort

| Level | Effort | Notes |
|-------|--------|-------|
| Junior (< 2 yr) | 10–15 person-days | Assumes single-site, ~50 devices, ~100 test cases; includes on-site execution, evidence capture, reporting, and customer coordination |
| Senior (5+ yr) | 5–8 person-days | Same scope; senior leverages efficient test execution and rapid defect triage |

Notes: SAT effort is primarily on-site, requiring travel and coordination with employer site access policies. Multi-site projects require separate SAT per site. Employer witness requirements may extend schedule (waiting for customer availability). Defect remediation effort is not included in SAT execution — it is tracked separately in the punch list. Complex integration environments (multiple employer systems, legacy protocols) may require 1.5–2× effort for integration verification.
