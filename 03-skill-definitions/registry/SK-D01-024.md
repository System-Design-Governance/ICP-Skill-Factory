## Description

Evaluate and score the cybersecurity risk posture of vendors, subcontractors, and component suppliers participating in the SuC delivery, covering vendor qualification, product security assessment, and ongoing monitoring obligations. This skill establishes the vendor security evaluation criteria, executes the qualification scoring process (per ID22 procurement procedure), and produces the vendor risk register that feeds into Gate 0 supply chain decisions. The boundary of this skill is vendor risk evaluation and scoring; the vendor management plan document that codifies the ongoing vendor relationship governance falls under SK-D01-031 ⏳, and the third-party component-level security verification (SBOM analysis, code review) falls under SK-D01-026 ⏳.

## Inputs

- Vendor information packages: company profiles, security certifications (ISO 27001, IEC 62443-4-1), product security documentation
- ICP Procurement and Supplier Management Procedure (ID22, Tier 3) — qualification scoring criteria and approved vendor list process
- Asset inventory with vendor-supplied components identified (from SK-D01-005)
- Customer/employer requirements for supply chain security (contractual obligations)
- IEC 62443-2-4 supply chain security requirements
- GOV-SD Gate 0 scope — supply chain risk is an explicit Gate 0 consideration

## Outputs

- Vendor Security Risk Assessment Report:
  - Vendor identification and scope of supply (hardware, software, services)
  - Security capability evaluation: certification status, secure development lifecycle maturity, vulnerability disclosure process, patch support commitment
  - Qualification scoring per ID22 criteria: quality system, delivery capability, technical competence, security posture (weighted scoring)
  - Risk rating per vendor: High / Medium / Low with justification
  - Compensating controls for vendors with identified gaps (contractual security requirements, inspection clauses, escrow agreements)
- Vendor Risk Register: per-vendor tracking with risk rating, review schedule, and escalation triggers
- Approved Vendor List (security-qualified subset): vendors meeting minimum security qualification threshold
- Vendor Security Requirements Specification: security clauses to be included in procurement contracts and subcontractor agreements
- Gate 0 Supply Chain Risk Summary: aggregated vendor risk input for Gate 0 decision package (per GOV-SD)

## Tools

- Vendor security questionnaire templates (based on IEC 62443-2-4 SP.01.01–SP.01.04 requirements)
- Procurement qualification scoring spreadsheet (per ID22 format)
- Vendor certification verification platforms (ISO/IEC accreditation body databases)
- Risk register tool (Excel or GRC platform)

## IEC 62443 Lifecycle Stages

- R0 (Pre-project — vendor qualification for new engagements)
- R1 (Security Requirements — vendor risk informs component selection and architecture decisions)

## Roles

- Security Engineering Role (GOV-SDP) — primary: defines vendor security evaluation criteria, executes security capability assessment
- PM (Project Manager) — coordinates vendor engagement, manages commercial aspects of vendor qualification
- SAC (Security Architect) — evaluates vendor product security implications for zone/conduit architecture
- Pre-Gate Design Support Role (GOV-SDP) — contributes vendor risk summary to Gate 0 decision package
- Head of System Design (GOV-SDP) — Gate 0 approval authority when vendor risk is significant

## Automation Potential

**Level:** Partial

Vendor questionnaire distribution and scoring aggregation can be automated through procurement platforms. Certification status verification can be partially automated through accreditation databases. However, security capability assessment requires expert judgment (evaluating SDL maturity, vulnerability disclosure practices), risk rating decisions involve contextual analysis (criticality of supplied components, availability of alternatives), and compensating control design requires human expertise.

## Acceptance Criteria

1. Every vendor supplying hardware, software, or security services to the SuC has a completed security risk assessment with documented risk rating (High/Medium/Low)
2. Qualification scoring completed per ID22 criteria for all vendors, with weighted scores recorded and threshold compliance documented
3. Vendor risk register maintained with review schedule: High-risk vendors reviewed quarterly, Medium annually, Low biannually
4. Vendor security requirements specification produced: contractual security clauses ready for inclusion in procurement documents
5. Gate 0 supply chain risk summary delivered as part of Gate 0 decision package (per GOV-SD)
6. No vendor with High risk rating without documented compensating controls and explicit approval by Security Engineering Role

## Estimated Effort

| Level | Effort | Notes |
|-------|--------|-------|
| Junior (< 2 yr) | 8–12 person-days | Assumes 5–10 vendors, initial qualification; includes questionnaire preparation, assessment, scoring, and report writing |
| Senior (5+ yr) | 4–6 person-days | Same scope; senior leverages existing vendor assessment templates and industry benchmarking |

Notes: Effort scales with vendor count and supply chain complexity. Projects with COTS-heavy architectures may require 1.5× effort due to product-level security analysis. Brownfield projects with existing vendor relationships require delta assessment only (reduced effort). Ongoing R4 vendor monitoring is approximately 1–2 person-days per quarterly review cycle.
