## Description

Automate engineering review workflows including review assignment and routing, checklist generation, finding tracking and resolution verification, and approval chain management. Review workflow automation accelerates design assurance processes while ensuring traceability and accountability. Automation eliminates manual routing, standardizes checklist application, and provides metrics on review status and finding closure. The skill integrates with document management systems (SK-D09-005) and design review processes (SK-D02-012: Design Review — Detailed; SK-D11-001: Design and Requirements Ownership). Workflow automation implements the governance and quality objectives defined in SK-D11-006 (Quality Plan Development) and SK-D11-010 (Internal Design Standards Maintenance).

## Inputs

- Design review procedures and checklist requirements (from SK-D11-006: Quality Plan Development)
- Organizational roles and approval authorities (from SK-D11-019 ⏳: Standards Ownership & Exception Arbitration)
- Design artifacts in standardized formats (documents, drawings, specifications to be reviewed)
- Document management system schema and API specifications (from SK-D09-005)
- Metadata for routing rules (document type, security level, project stage, priority)
- Historical review data (typical review duration, finding severity distributions, closure timeline)
- Compliance and audit requirements (logging, approval evidence, change tracking)

## Outputs

- Review workflow orchestration specification:
  - Document intake and classification (automatic detection of document type and applicable review scope)
  - Reviewer assignment rules (based on expertise, availability, conflict-of-interest checks)
  - Checklist generation and customization (project-specific or standard templates)
  - Finding tracking and lifecycle management (creation, assignment, resolution verification, closure)
  - Approval chain definition and signature/consent capture
- Workflow automation implementation (scripts, plugins, or cloud workflow service)
- Integration layer with document management system (API clients, data synchronization)
- Finding and finding resolution tracking system:
  - Finding classification (by category, severity, responsibility area)
  - Assignment and escalation rules
  - Closure verification procedures (evidence capture, review sign-off)
  - Metrics and SLA tracking (review cycle time, finding closure time)
- Workflow dashboard and reporting specification (review status, bottlenecks, SLA compliance)
- Change management and audit logging (all workflow state transitions logged with audit trail)
- User training and communication plan for review teams

## Tools

- Workflow orchestration (Airflow, Prefect, or cloud workflow engines)
- Document management system API (REST, SDK if available)
- Checklist and form tools (Typeform, Google Forms, or custom web forms)
- Finding/issue tracking (Jira, Azure DevOps, or internal ticketing system)
- Approval workflow engines (custom workflow state machine or third-party approval service)
- Notification and communication (email, Slack, Microsoft Teams for reviewer alerts)
- Reporting and analytics (dashboards, SQL queries for SLA tracking)
- Audit logging and compliance tools (database logs, change tracking)

## Roles

- SYS (System Architect) — primary: designs review workflow, defines routing and approval rules
- QA (Quality Assurance) — execution: operates workflow, manages finding tracking, produces metrics
- PE/O (Project Engineer/Owner) — approves review workflow, signs off on design deliverables
- Process Admin — supports: maintains workflow configuration, user access, audit logs

## Dependencies

### Hard Dependencies (Required Before Execution)

- SK-D11-006: Quality Plan Development — review procedures and checklist definitions must be finalized
- SK-D09-005: Document Management System — workflow integration requires DMS functionality for document access and metadata
- Organizational approval authorities and role definitions — workflow cannot be automated without clear routing rules

### Soft Dependencies (Enhances Effectiveness)

- SK-D02-012: Design Review — Detailed — understanding review process ensures automation supports all required review activities
- SK-D11-001: Design and Requirements Ownership — ownership assignments drive reviewer notification and accountability
- SK-D13-003: AI-Assisted Design Review — automated technical review findings can feed into review workflow for engineer remediation

## Acceptance Criteria

1. Workflow specification documents review stages, decision points, approval authorities, and finding escalation paths
2. Document intake correctly classifies document type and applicable review scope (100% accuracy on test set)
3. Reviewer assignment routing routes documents to correct reviewers per assignment rules; manual override allowed with audit trail
4. Checklist generation produces complete project-specific checklists with all applicable items from SK-D11-006 quality plan
5. Finding creation, assignment, and tracking functional: findings linked to document, reviewer, affected artifact; closure verification enforced
6. Approval chain captures evidence of review (timestamp, reviewer name, approval status); audit trail complete for compliance
7. Workflow and reporting dashboard functional: displays review status (pending, in-review, approved), bottlenecks, SLA metrics
8. Integration with document management system tested: documents automatically appear in workflow; review completions reflected in DMS status
9. Workflow tested end-to-end with representative design package; minimum 3 distinct project scenarios (low/medium/high complexity)
10. User training and runbook completed; workflow approved by SYS, QA, and PE/O

## Estimated Effort

| Level | Effort | Notes |
|-------|--------|-------|
| Junior (< 2 yr) | 20–30 person-days | Includes workflow design, automation implementation, DMS integration, testing; requires learning orchestration frameworks |
| Senior (5+ yr) | 12–18 person-days | Same scope; leverages prior workflow automation experience and organizational knowledge of review practices |

Notes: Simple linear review workflows (author → reviewer → approver) require 15–20 person-days. Complex multi-level approval chains with conditional routing or parallel reviews may add 8–12 person-days. DMS integration complexity (API availability, data synchronization) is primary variability.
