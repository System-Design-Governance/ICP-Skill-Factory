## Description — Purpose & Scope
Build and maintain the organizational technical knowledge base containing design patterns, reference architectures, solved problem databases, vendor-specific configuration guides, and troubleshooting procedures. This skill ensures technical knowledge is systematically organized, discoverable, and continuously enriched with lessons learned and design standards to support consistent, high-quality engineering delivery.

## Roles
- Design and implement technical knowledge base structure and taxonomy
- Develop and maintain design pattern library with implementation examples
- Create and curate reference architecture documentation
- Build solved problem database with troubleshooting procedures
- Document vendor-specific configuration guides and best practices
- Establish content creation and submission workflows
- Implement search and discovery optimization
- Define and manage access control policies
- Integrate lessons learned from SK-D11-008 into knowledge base
- Incorporate design standards from SK-D11-010
- Maintain version control and change history tracking
- Monitor knowledge base usage metrics and effectiveness

## Roles — Competency Requirements
- Deep technical expertise across engineering domains
- Proficiency in knowledge base systems and information architecture
- Strong technical writing and documentation skills
- Experience with taxonomy design and knowledge organization
- Understanding of reference architecture development
- Expertise in design patterns and reusable solution design
- Vendor platform and configuration knowledge
- Search and discovery system optimization skills
- Experience with access control and security policies

## Acceptance Criteria
- Comprehensive knowledge base structure documented and implemented
- Minimum 50+ reference architectures documented and maintained
- Complete design pattern library with code examples and rationale
- Solved problem database covering 80%+ common technical issues
- Vendor-specific guides for all supported platforms and products
- Content submission and review workflow established and active
- Knowledge base search performance optimization (< 1 second queries)
- Documented integration points with SK-D11-008 and SK-D11-010
- Minimum 70% engineering team adoption rate
- Quarterly knowledge base audits and content refresh cycles
- Usage analytics demonstrating value and impact
