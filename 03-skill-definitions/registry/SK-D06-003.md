## Description

Develop comprehensive terminal block schedules documenting all terminal blocks installed in control panels and instrument cabinets. A terminal block schedule is a structured table listing every terminal block, the wires connected to each terminal (source/destination wire number), cable references, signal type (analog, digital, power, ground, etc.), and voltage level. The schedule serves as the authoritative reference for panel fabrication personnel, field installation teams, and maintenance technicians, and is essential for ensuring wiring integrity, enabling troubleshooting, and supporting future modifications. This skill requires careful coordination with the functional schematic (SK-D04-001 ⏳), wiring diagrams (SK-D06-002), and terminal block physical layout in fabrication drawings (SK-D06-004). Terminal block schedules must be clear, unambiguous, and suitable for non-engineering personnel to follow during assembly and installation.

## Inputs

- Functional electrical schematic diagrams (from SK-D04-001 ⏳: Functional Electrical Schematic Design)
- Wiring diagrams with wire numbers (from SK-D06-002: Wiring Diagram Development; or developed iteratively)
- Control panel layout and terminal block arrangement (from SK-D06-004: Fabrication Drawing Production)
- Equipment datasheets and I/O specifications (relay modules, drives, I/O modules, sensors, actuators)
- Cable schedule and inter-panel connection matrix (from SK-D02-003 ⏳: Interface Control Document or SK-D07-001: Interface Integration Matrix Development)
- Customer or organizational terminal block labeling standards

## Outputs

- Terminal Block Schedule (structured table format, PDF and Excel):
  - Terminal block ID (panel ID + block number, e.g., "PLC-TB-01")
  - Terminal ID within block (e.g., "1", "2", ..., "12")
  - Wire number (source wire from wiring diagram)
  - Wire source (e.g., "PLC output X1", "Remote switch S-001", "Ground rail")
  - Wire destination (e.g., "Motor relay K-01", "Indicator light L-01")
  - Cable reference (if wire is part of a multi-conductor cable)
  - Signal type (digital input, digital output, analog 4–20mA, power 24VDC, ground, etc.)
  - Voltage/current rating (e.g., "24 VDC 5A", "4–20mA 12-bit ADC")
  - Shield/armor information (if applicable)
  - Notes/special handling (e.g., "shielded, keep away from power", "twisted pair required")
- Cross-reference matrix linking terminal block schedule to wiring diagram wire numbers (for verification)
- Panel terminal block layout diagram (visual representation of physical terminal block positions within panel)
- Revision-controlled schedule with change tracking per SK-D09-005

## Tools

- Excel / structured markdown (primary; human-readable tabular format suitable for panel shop and field teams)
- AutoCAD Electrical or EPLAN (automated terminal block schedule generation from schematic and wiring diagram)
- Google Sheets / Airtable (collaborative updates and version control)

## Standards

- IEC 61346: Structured Principles and Reference Designations for Electric Devices and Systems — terminal labeling and reference designations
- IEC 60950-1 / IEC 62368-1: Safety of electrical equipment for measurement, control, and laboratory use — terminal block electrical safety ratings
- IEC 60617: Graphical symbols — schematic representation of terminal blocks
- IEC 61810-1: Electromechanical elementary relays — wire and terminal marking conventions
- Customer electrical standards and site-specific terminal block formats

## IEC 62443 Lifecycle Stages

- R2 (Detailed Design) — terminal block schedule is a key element of detailed design documentation required for fabrication, installation, and commissioning

## Roles

- DES (Design Engineer) — primary: develops and maintains terminal block schedule, ensures cross-reference accuracy with wiring diagrams and schematics
- Fabrication/Panel Integration Engineer — uses schedule for panel assembly; provides feedback on physical layout constraints
- Field Installation/Commissioning Engineer — uses schedule for field wiring verification and commissioning

## Dependencies

### Hard Dependencies (Required Before Execution)

- SK-D04-001 ⏳: Functional Electrical Schematic Design — terminal block assignments must derive from the approved functional schematic

### Soft Dependencies (Enhances Effectiveness)
- SK-D06-002: Wiring Diagram Development — terminal block schedule cross-references wire numbers (developed iteratively)

- SK-D06-004: Fabrication Drawing Production — physical terminal block layout in the fabrication drawing informs the order and grouping of terminal block entries in the schedule
- SK-D06-005: Wire Sizing Calculation — wire gauge and cable type information enriches the terminal block schedule for field assembly guidance
- SK-D02-003 ⏳: Interface Control Document — inter-panel cable references must be coordinated with the terminal block schedule

## Acceptance Criteria

1. Every terminal block in the approved control panel layout (SK-D06-004) has an entry in the schedule with a unique terminal block ID
2. Every terminal in every terminal block is explicitly listed with: terminal ID, source wire number, destination wire number, signal type, and voltage rating
3. 100% of wires appearing in SK-D06-002 (Wiring Diagram) are cross-referenced to corresponding terminal block entries (traceability matrix complete)
4. No terminal block entry references a wire number that does not exist in the wiring diagram (consistency check passes)
5. Signal types and voltage ratings are consistent with equipment datasheets and IEC standards
6. Cable references for inter-panel or field cables are complete and match SK-D02-003 ⏳ (Interface Control Document)
7. The schedule is formatted clearly enough for non-engineering personnel (panel shop, field technicians) to follow during assembly and installation
8. Schedule has been reviewed and approved by DES lead and reviewed by the Fabrication/Integration Engineer for constructability

## Estimated Effort

| Level | Effort | Notes |
|-------|--------|-------|
| Junior (< 2 yr) | 3–5 person-days | Single panel or modular subsystem (~50–100 I/O terminals); includes data entry, cross-referencing, and verification |
| Senior (5+ yr) | 1–2 person-days | Same scope; senior can leverage automated schedule generation tools and cross-reference macros |

Notes: Effort scales with number of terminal blocks and I/O points. Multi-panel systems with inter-panel cabling may require 1.5–2× effort. Revisions to the wiring diagram require corresponding updates to the terminal block schedule.
