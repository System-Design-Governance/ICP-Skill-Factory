## Description

Develop the comprehensive SCADA point list defining all monitored and controlled points including: point ID, description, data type, scaling factors, engineering units, alarm limits, scan rate, and data source mapping. The point list serves as the master data reference for SCADA system configuration, defining the information model that bridges power system field devices to EMS/SCADA/DERMS control platforms. Results feed into SCADA database structure design (SK-D05-002), alarm threshold definition (SK-D05-003), and IEC 61850 mapping (SK-D05-010) for digital substations.

## Inputs

- One-line diagram and network topology (substations, circuits, equipment locations)
- Equipment specifications (transformers, circuit breakers, disconnects, capacitor banks, DERs)
- Instrumentation schedule (PT/CT ratings, RTU/PMU point counts, analog/digital I/O counts)
- Protection logic diagrams (trip signals, alarm outputs from relays/IEDs)
- Operational procedures and control requirements (load flow targets, voltage regulation setpoints)
- SCADA architecture design (RTU/FEP configuration, polling intervals, data retention policy)
- Regulatory and utility standards (IEEE 1379, NERC CIP data security requirements)
- Historical SCADA configurations (legacy point lists for migration planning)

- Scan rate and priority classification (real-time, near-real-time, historical)

- Point list validation report (completeness, coverage, consistency checks)
- IEC 61850 Logical Node cross-reference (for digital substation mapping)
- SCADA database import scripts or configuration files

## IEC 62443 Lifecycle Stages

- **Planning & Requirements**: Define monitored/controlled point requirements and data flow needs
- **Design & Implementation**: Develop master point list; configure RTU/FEP polling and data routing
- **Verification & Validation**: Validate point list completeness and SCADA database integration
- **Operations & Maintenance**: Manage point list changes; track point additions/retirements
- **Decommissioning**: Archive point list; purge retired points and associated historical data

## Acceptance Criteria

1. Point list includes all monitored field devices: analog inputs (voltage, current, temperature, pressure), digital status (breaker position, alarm flags), and controlled outputs (breaker trip, tap changer commands)
2. Every point has unique, descriptive ID following enterprise naming convention (e.g., SUB001_LINE02_CURR_RMS)
3. Data type specified for all points (16-bit/32-bit integer, floating-point, enumeration, string)
4. Scaling factors and engineering units documented (MVA, kV, A, °C, percentage, ppm, etc.)
5. Alarm limits defined for all analog points (high/low alarm thresholds, rate-of-change limits where applicable)
6. Scan rates assigned with documented rationale (critical protection signals: 100 ms; control points: 500 ms; status: 5 sec; historical: 60 sec+)
7. Data source mapping complete: RTU ID, coil/register address, communication protocol (Modbus RTU/TCP, DNP3, IEC 60870-5-104)
8. IEC 61850 Logical Node cross-reference provided (e.g., MMXU for metering points, XCBR for breakers)
9. Point count validation: total points within RTU/FEP capacity with 20% headroom for expansion
10. Sign-off by System Architect, SCADA Engineer, and Operations Team
