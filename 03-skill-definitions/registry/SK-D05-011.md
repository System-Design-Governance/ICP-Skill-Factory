## Description

Configure OPC UA (OLE for Process Control Unified Architecture) servers and clients for secure data exchange between IT and OT systems in industrial automation and energy management environments. This skill covers OPC UA endpoint configuration on field devices (PLCs, gateways, meters), server information model design (object hierarchy, variable definitions, method signatures), security policy selection and enforcement (Basic256Sha256, Aes128_Sha256_RsaOaep, certificate management), user authentication and authorization (user tokens, roles, permissions), endpoint discovery mechanisms, and client-side configuration for enterprise SCADA, MES (Manufacturing Execution Systems), or cloud gateways. OPC UA provides the primary IT/OT integration protocol per SK-D02-005 (IT/OT Integration and Data Contextualization) and respects Zone/Conduit security boundaries from SK-D01-001. This skill is essential for Industry 4.0 and advanced grid-level analytics and feeds into system commissioning, cybersecurity validation, and operational data exchange.

## Inputs

- Approved Data Flow Diagram and IT/OT data requirements (from SK-D02-004 ⏳: Data Flow Diagram Development, SK-D02-005 ⏳: IT/OT Integration)
- OPC UA information model specification: data points, object hierarchy, method signatures (from SK-D02-005 ⏳)
- Zone/Conduit security boundaries and authentication/encryption policies (from SK-D01-001: Zone/Conduit Architecture Design)
- Cybersecurity control selection and certificate infrastructure (from SK-D01-003 ⏳: Cybersecurity Control Selection; certificate policy from SK-D01-001)
- OPC UA server platform specification (Kepware, Wonderware, National Instruments, vendor-specific gateway)
- Client application requirements (enterprise SCADA platform, MES, cloud gateway)
- Enterprise Active Directory / LDAP configuration (for user authentication)
- Network topology and firewall rules (from SK-D02-001 ⏳: OT Network Topology Design)

## Outputs

- OPC UA Server Information Model Definition (.xml per OPC UA specification): object types, variables, methods, event structures
- OPC UA Endpoint Configuration Specification: endpoint URLs, security policies, user authentication methods, access control lists
- Security Policy and Certificate Management Plan: certificate hierarchy, root/intermediate/leaf certificate assignments, renewal procedures
- User and Role Definition Document: user accounts, role-based access control (RBAC) mappings, permission assignments
- OPC UA Client Configuration Specification: endpoint discovery, server URL assignment, security policy and certificate configuration per client
- Firewall and Network Access Rule Specification: port assignments, IP address allowlists/denylists, VPN/encrypted tunnel requirements
- OPC UA Configuration Validation Report: connectivity testing, security policy enforcement validation, performance baseline
- IT/OT Integration Testing Plan: data point read/write test cases for commissioning and validation

## Acceptance Criteria

1. OPC UA information model (.xml) is complete with all required data points from approved Data Flow Diagram; model passes OPC UA schema validation
2. Server endpoints are configured with approved security policies (at minimum Basic256Sha256 per IEC 62443-3-3 for Zone 1/Zone 2 communication)
3. User authentication is configured with credentials from enterprise directory (Active Directory, LDAP) or local user tokens with documented access control lists
4. Role-based access control (RBAC) is implemented: roles defined per Zone/Conduit zone assignment, read/write permissions explicitly assigned per role
5. Certificates (root, intermediate, leaf) are generated and installed per certificate management plan; certificate chain is complete and validated
6. Firewall rules and network access controls are configured per specification; firewall rule changes are documented and approved
7. Client endpoints are discovered and connected successfully; data read/write operations function within latency requirements
8. Security validation testing confirms encryption is active (TLS 1.2 or higher), authentication is enforced, and unauthorized access is denied
9. OPC UA configuration documentation is approved by DES and reviewed by STC for security compliance
