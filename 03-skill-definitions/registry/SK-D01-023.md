## Description

Design and implement the malware protection framework for all endpoint categories within the SuC, covering three deployment scenarios: on-site host computers (SCADA HMI servers), temporarily connected devices (engineering laptops), and portable media (USB drives, external storage). This skill establishes the centralized antivirus management architecture, deploys endpoint protection agents, configures definition update mechanisms, and defines incident management procedures for malware detection events. The boundary of this skill is malware protection design and deployment; the broader endpoint hardening that includes service minimization and port disabling falls under SK-D01-019, and the incident response procedure activated upon malware detection falls under SK-D01-016.

## Inputs

- Asset inventory with OS platforms and network connectivity (from SK-D01-005)
- Zone/Conduit architecture (from SK-D01-001) — determines definition update distribution paths and central management connectivity
- FR/SR mapping for malicious code prevention (from SK-D01-007: SR 3.2 Malicious Code Protection per IEC 62443-3-3)
- Antivirus product selection decision (e.g., Palo Alto Cortex XDR per ID12 exemplar)
- Network architecture for definition update distribution (air-gapped vs. connected update paths)
- Customer requirements for portable media and temporarily connected device policies

## Outputs

- Malware Protection Implementation Document (per ID07 §7.0 exemplar):
  - General rules: systematic approach to malware protection in OT environments with risk assessment of antivirus impact
  - Central management architecture: centralized console deployment, policy enforcement, real-time threat visibility
  - Protection per deployment scenario:
    - On-site host computers: agent installation, configuration, verification workflow
    - Temporarily connected devices: pre-connection validation, protection enforcement
    - Portable media: scan-before-use policy, approved media list, access restrictions
  - Update & patch mechanisms: definition file update channels per scenario (separate from OS patches), automatic/scheduled update configuration
  - Incident management: detection → alerting → isolation → remediation → recovery → post-incident analysis
  - Devices with no protection mechanisms: exception documentation with compensating controls
- Antivirus Agent Deployment Report: per-device installation verification (per ID12 §4.1 exemplar):
  - Installation process success confirmation
  - System function integrity verification (no adverse effects on OT operations)
  - Agent-to-controller connectivity verification (IP configuration, server connection)
  - Endpoint protection activation confirmation
- Logical Malware Protection Architecture Diagram: agent distribution, central management console, definition update paths, monitoring data flows
- Exception Register: devices where antivirus cannot be installed with compensating controls and approval

## Tools

## Automation Potential

**Level:** Partial

Agent deployment to Windows-based systems can be automated through centralized management consoles (push deployment). Definition updates are automated once channels are configured. Monitoring and alerting are automated. However, antivirus impact assessment on OT system operations requires manual testing per device, exception evaluation for devices that cannot run agents requires human judgment, and incident response upon detection requires human decision-making.

## Acceptance Criteria

1. Malware protection deployed on 100% of capable devices in the asset inventory; exceptions documented with compensating controls
2. Agent installation verified per device: system function integrity confirmed (no adverse effects on OT operations per ID12 verification workflow)
3. Central management console operational: real-time visibility of all protected endpoints, policy enforcement active
4. Definition update mechanism configured and tested: automatic or scheduled updates verified for all deployment scenarios
5. Portable media policy enforced: scan-before-use procedure documented and scanning infrastructure operational
6. Exception register complete: all devices without agent protection have documented compensating controls approved by Security Engineering Role

## Estimated Effort

| Level | Effort | Notes |

| Junior (< 2 yr) | 10–15 person-days | Assumes ~50 devices, Palo Alto deployment; includes agent installation, verification per device, central management setup, and procedure documentation |
| Senior (5+ yr) | 5–8 person-days | Same scope; senior leverages push deployment and templated verification procedures |

Notes: Air-gapped environments require additional effort for offline definition update mechanisms (sneakernet or dedicated update stations). IACS embedded devices that cannot run traditional antivirus require compensating control design (1–2 additional days). Multi-vendor OT environments may require multiple protection solutions.
