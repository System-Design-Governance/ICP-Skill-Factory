## Description

Design Virtual Power Plant dispatch algorithms that optimize Distributed Energy Resource (DER) allocation based on market signals, grid operating constraints, contractual service obligations, and real-time asset availability. Covers objective function definition (revenue maximization, cost minimization, ancillary service provision), constraint modeling (power balance, voltage/frequency limits, ramp rates, asset health), and real-time dispatch logic for implementation within Energy Management System (EMS) and Distributed Energy Resource Management System (DERMS) platforms. This skill bridges power system operations and cybersecurity by ensuring that dispatch logic is resilient, auditable, and integrated with security control strategies (e.g., load shedding prioritization, critical asset protection during contingencies).

## Inputs

- Power system model and network constraints (from SK-D03-010: Power System Modeling)
- DER asset inventory with technical specifications:
  - Power rating, energy capacity, ramp rates, min/max output
  - Response time, control modes (voltage/frequency droop, active power-frequency, reactive power-voltage)
  - Degradation models and state-of-health parameters
  - Geographic location and interconnection point
- Load forecast and demand curves (hourly, 15-minute resolution, seasonal)
- Electricity market data (spot prices, ancillary service prices, reserve requirements)
- Grid operating constraints:
  - Voltage and frequency limits at each bus
  - Transmission/distribution line thermal limits
  - Feeder voltage drop constraints
  - System protection settings (relay coordination)
- Contractual obligations (power purchase agreements, grid support service contracts)
- Zone/Conduit architecture and security priority hierarchy (from SK-D01-001)
- Historical operational data and contingency scenarios

## Outputs

- VPP Dispatch Algorithm Specification Document including:
  - Objective function(s) with mathematical formulation and weights
  - Decision variables (DER power setpoints, storage charging/discharging, load control)
  - Constraint set with inequalities and bounds
  - Time-domain discretization and forecast horizon
  - Boundary conditions and assumptions
- Real-time Dispatch Logic Flowchart:
  - Measurement and data acquisition
  - Forecasting module integration
  - Constraint evaluation
  - Optimization execution and convergence criteria
  - Setpoint generation and communication to assets
  - Exception handling and fallback logic
- DERMS/EMS Configuration Specification:
  - Parameter tables (asset limits, droop gains, reserve margins)
  - Communication protocol and latency requirements
  - State machine logic (normal operation, contingency response, grid disturbance modes)
  - Monitoring and alerting logic for constraint violations
- Validation Report:
  - Performance metrics (revenue/cost, constraint feasibility, computation time)
  - Scenario testing results (peak load, low renewable generation, forced outages)
  - Security analysis (resilience to sensor failure, communication delay, market price manipulation)
- Implementation Roadmap for EMS/DERMS deployment

## Acceptance Criteria

1. Objective function explicitly defined with mathematical formulation and all decision variables identified
2. Constraint set covers 100% of grid operating limits (voltage, frequency, thermal, ramp rate) with documented sources
3. DER asset parameters validated against manufacturer datasheets and field commissioning tests
4. Dispatch algorithm tested on minimum 10 representative load/generation scenarios (peak, off-peak, low-wind, high-solar, contingency)
5. All constraints remain feasible under normal and N-1 contingency conditions; infeasibility cases documented with mitigation strategy
6. Real-time computation time measured and shown to be < 1 minute for operational use case (typically << 1 minute for 15-minute dispatch interval)
7. Dispatch logic includes security control points: asset prioritization, load shedding hierarchy, frequency support sequencing
8. Algorithm design document reviewed and approved by SYS, PRAC, and SEC

## Estimated Effort

| Level | Effort | Notes |
|-------|--------|-------|
| Junior (< 2 yr) | 20–30 person-days | Assumes single-objective optimization, 5–10 DER assets, medium complexity power system; includes algorithm development, validation, documentation |
| Senior (5+ yr) | 10–15 person-days | Same scope; senior leverages optimization libraries, prior dispatch designs, and platform expertise; faster convergence analysis |

Notes: Multi-objective dispatch (revenue + emissions + resilience) or large DER fleet (50+ assets) may require 1.5–2× effort. Market-integrated algorithms with price forecasting add 5–10 days. Non-convex problems (e.g., unit commitment with startup costs) may require heuristic methods and 2–3× baseline effort.
