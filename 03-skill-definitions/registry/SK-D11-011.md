## Description

Develop and maintain the engineering competency framework that defines role-based capability requirements, proficiency levels, assessment criteria, and KPI structures for all engineering roles within the System Design Department. Under ICP governance (GOV-SDP), the department operates with 7 defined functional roles, each with SMART KPI targets, RACI assignments across all gate activities, and a scoring formula (RCW/AF/PVF) that links competency evidence to performance evaluation. This skill establishes the framework itself — the role definitions, competency matrices, proficiency ladders, and assessment instruments. Security competency (per IEC 62443) is the primary instantiation, but the framework is generalized to cover all engineering disciplines per ADR-006. The boundary of this skill is framework design and maintenance; the actual training program that develops competencies falls under SK-D11-012, and the periodic KPI evidence collection and scoring falls under SK-D11-021 ⏳.

- IEC 62443-2-4 SP.01.01–SP.01.03: Service provider staffing and qualification requirements

## Outputs

- Engineering Competency Framework Document:
  - Role Taxonomy: 7 functional roles with clear role boundaries
    - Head of System Design (department leadership, gate authority)
    - Security Architect (SAC — zone/conduit design, SL decision)
    - System Architect (SYS — system topology, integration architecture)
    - Security Engineering Role (implementation, SR verification)
    - Design Engineer (DES — detailed design, configuration)
    - Security Technical Consultant (STC — protocol, encryption specialization)
    - Design QA Role (quality assurance, traceability verification)
    - Pre-Gate Design Support (feasibility, CBOM, concept architecture)
  - Per-role Competency Matrix: technical competencies, domain knowledge, tool proficiency, soft skills — each with proficiency levels (Foundational / Practitioner / Expert)
  - RACI Matrix: per-role responsibility assignment across all gate activities (Gate 0–3) and lifecycle phases (R0–R5)
  - KPI Structure: per-role SMART KPIs (6+ per role) with measurement method, target threshold, evidence requirements
  - Proficiency Assessment Instruments: evaluation criteria, scoring rubrics, evidence collection templates
- Role-to-Skill Mapping: which skills from the ICP Skill Factory each role is expected to perform (linking competency framework to skill registry)
- Competency Gap Analysis Template: assessment tool for evaluating individual engineers against the framework
- Annual Review Cycle Specification: data freeze dates, assessment windows, settlement process (per GOV-SDP annual governance cycle)

## Automation Potential

**Level:** Partial

Competency matrix templates and RACI generation can be automated from role definitions. KPI tracking dashboards can be automated (data collection from project systems, scoring computation). Gap analysis scoring can be partially automated (assessment scores aggregated against proficiency thresholds). However, role boundary definition requires organizational judgment, KPI target setting requires strategic alignment with department goals, proficiency level calibration requires expert consensus, and individual assessment interpretation requires human evaluation context.

## Acceptance Criteria

1. Framework covers all 7 GOV-SDP functional roles with complete role definitions: job description, scope boundary, reporting line, and gate-specific responsibilities
2. Per-role competency matrix includes minimum 8 competency items spanning technical, domain, tool, and soft skill categories — each with 3 proficiency levels and assessment criteria
3. RACI matrix covers all gate activities (Gate 0–3) and lifecycle phases (R0–R5): every activity has exactly one Responsible and one Accountable role; no activities without role assignment
4. Per-role KPI structure includes 6+ SMART KPIs with: measurement method, target threshold, evidence type, and collection frequency — no KPIs with subjective-only measurement
5. RCW/AF/PVF scoring formula documented with: calculation method, input variables, weighting rules, and safeguard mechanisms (per GOV-SDP)
6. Annual review cycle specification includes: data freeze date, assessment window, settlement process, and CRM mapping — aligned with GOV-SDP annual governance cycle

Security competency (IEC 62443 specialization) is the most detailed instantiation and should be developed first as the template for other discipline competencies. ID21 QP-02 provides the organizational training procedure that the framework must align with.
