## Description

Execute the secure decommissioning of OT/ICS systems from active operation per IEC 62443 lifecycle requirements (ID01 §6.5.3), ensuring no residual security vulnerabilities remain after system removal. This skill encompasses the complete decommissioning lifecycle including data sanitization, credential revocation, certificate destruction, license deactivation, physical equipment removal, and documentation archiving. The boundary of this skill begins after formal decommissioning approval (from SK-D10-005: Decommissioning Planning) and ends when verification confirms zero residual security artifacts. Critical infrastructure must maintain operational continuity throughout the decommissioning process, requiring careful sequencing of removal steps and rollback planning.

## Inputs

- Approved decommissioning plan with sequencing schedule (from SK-D10-005: System Decommissioning Planning)
- Current system configuration documentation and asset inventory (from SK-D01-005: Asset Inventory Development)
- Active credential and certificate registry for the target system
- Vendor contract and license inventory for affected components
- Risk assessment for system removal impact (from SK-D01-007: Detailed Risk Assessment)
- Network dependency mapping showing interconnections with remaining systems (from SK-D02-004: Data Flow Diagram Development)

## Outputs

- Data Destruction Certification Report: media type, destruction method (per NIST SP 800-88), verification evidence, chain of custody
- Credential and Certificate Revocation Log: revoked accounts, expired certificates, destroyed keys, timestamped confirmation
- Physical Equipment Removal Record: equipment ID, disposal method, receiving party, environmental compliance
- System Archive Package: final configuration snapshots, operational history, maintenance records for long-term retention
- Post-Decommissioning Security Verification Report: scan results confirming zero residual access vectors
- Updated asset inventory reflecting removed systems

## Tools

- Data destruction tools (DBAN, hardware degaussers, physical shredders for storage media)
- Certificate management systems (PKI, Active Directory Certificate Services)
- Network scanning tools (Nmap, Nessus) for post-decommissioning verification
- CMDB / asset management systems for inventory updates
- Secure archival storage systems for documentation retention

## Standards

- IEC 62443-2-1: Security Management System — decommissioning procedures

- NIST SP 800-88 Rev. 1: Guidelines for Media Sanitization — data destruction methods and verification
- NIST SP 800-82 Rev. 3: Guide to OT Security — lifecycle management guidance
- Local regulatory requirements for hazardous material disposal and data privacy

## IEC 62443 Lifecycle Stages

- R5 (Decommissioning — execute secure system removal, data destruction, and credential revocation per approved plan)

## Roles

- PM (Project Manager) — primary: coordinates decommissioning execution sequence, manages stakeholder communication
- SAC (Security Architect) — verifies all security artifacts are removed, conducts post-decommissioning security assessment
- OPS (Operations Engineer) — executes physical removal and data destruction procedures
- STC (Security Technical Consultant) — manages credential revocation and certificate destruction

## Dependencies

### Hard Dependencies (Required Before Execution)

- SK-D10-005: System Decommissioning Planning — approved decommissioning plan with sequencing and rollback procedures is prerequisite
- SK-D01-005: Asset Inventory Development — accurate asset inventory needed to ensure complete removal

### Soft Dependencies (Enhances Effectiveness)

- SK-D10-004: System Lifecycle Review — lifecycle review informs decommissioning priority and timing
- SK-D01-007: Detailed Risk Assessment — risk assessment quantifies impact of incomplete decommissioning
- SK-D02-004: Data Flow Diagram Development — data flow mapping identifies all system interconnections requiring disconnection

## Acceptance Criteria

1. All sensitive data on decommissioned systems is irrecoverably destroyed with documented verification per NIST SP 800-88 standards
2. 100% of system credentials, certificates, and cryptographic keys are revoked and confirmed non-functional
3. Post-decommissioning network scan confirms zero residual access vectors from removed systems
4. Physical equipment is securely removed and disposed per applicable regulatory and environmental requirements
5. Complete system archive package is produced and stored in designated long-term retention facility
6. Updated asset inventory accurately reflects the removal of all decommissioned components
7. All vendor licenses and support contracts for decommissioned systems are deactivated with documented confirmation
