## Description

This skill enables the Head of System Design to manage personnel security qualifications throughout the project lifecycle. It encompasses background verification, security clearance tracking, certification maintenance, and role-based access authorization for project team members in accordance with ID21 (Personnel Security Procedure) and ID06 §5.4 (Personnel Security Requirements), ensuring that personnel assigned to security-sensitive roles meet qualification requirements before accessing classified systems or information.

## Standards

IEC 62443-2-1 (Security Management System — Personnel Security), IEC 62443-3-3 (System Security Requirements — Personnel Access Control), ID21 (Organizational Personnel Security Procedure — Tier 3), ID06 §5.4 (Personnel Security Requirements in Classified Environment Access).

## IEC 62443 Lifecycle Stages

R0 (Planning), R1 (Requirements), R2 (Design), R3 (Implementation), R4 (Verification), R5 (Operations)

## Acceptance Criteria

### Background Verification Process
- Background verification requirements are defined per role classification and access authority level (standard, elevated, critical infrastructure).
- Background verification checklist includes identity verification, employment history verification, criminal record check, credit history review (where applicable), and reference checks.
- Background verification is conducted by authorized personnel with documented evidence collection and retention per organizational records policy.
- Background verification results are documented with clear pass/fail determination and any conditional clearance limitations.
- Background verification is refreshed at defined intervals (e.g., annually for elevated access, biannually for critical infrastructure roles).

### Security Clearance Tracking
- Security clearance levels are assigned per organizational security classification: Level 1 (Standard Access), Level 2 (Elevated Access), Level 3 (Critical Infrastructure Access).
- For each team member, clearance status is tracked with clearance level, issue date, expiration date, and responsible clearance authority.
- Clearance tracking is maintained in secure personnel database with access limited to authorized management.
- Personnel are notified of approaching clearance expiration with re-certification requirements defined in advance.
- Clearance status is verified prior to assignment to security-sensitive roles or access to classified information.

### Certification Maintenance
- Security-relevant certifications are identified per role (e.g., IEC 62443 security engineering, CISSP, approved security training programs).
- Certification status is tracked with certification type, issuing authority, issue date, and expiration date.
- Certification renewal requirements and timelines are monitored with advance notice to personnel.
- Personnel assigned to security-critical roles maintain current certifications throughout project lifecycle.
- Certification gaps are identified and development plans established per SK-D11-012 (Competency Gap Remediation).

### Role-Based Access Authorization
- Role-based access control (RBAC) matrix is defined mapping project roles to system access rights, information classification access, and facility access.
- RBAC matrix includes distinguishing access for: design/analysis (read-only or limited modification), implementation/verification (full access), and operational/support roles.
- Access authorization requests are submitted with role justification and security clearance verification.
- Access is provisioned only after authorization approval and personnel completion of role-specific security training.
- Access rights are reviewed quarterly to ensure continued alignment with assigned role and project responsibilities.

### Security-Sensitive Role Assignment Control
- Security-sensitive roles are identified per ICP project (e.g., Security Architect, Threat Modeler, Countermeasure Verifier, Information Security Manager).
- Assignment to security-sensitive roles requires documented approval confirming: clearance level sufficient for role, relevant certifications current, background verification passed, and role-specific security training completed.
- Assignment records include effective date, assigned responsibilities, and access scope.
- Termination of security-sensitive role assignments is documented with timely access revocation and knowledge transfer completion.

### Classified Information & System Access Control
- Personnel accessing classified information or critical systems must have appropriate security clearance confirmed prior to access grant.
- Access to classified information is logged and periodically audited for appropriateness.
- Non-disclosure agreements (NDAs) and information handling policies are signed before access to classified information is granted.
- Security debriefing is conducted upon termination of classified information access or role.

### Personnel File Maintenance & Documentation
- Personnel security file is established for each team member with background verification results, clearance status, certifications, training records, and access authorization.
- Personnel security files are maintained confidentially per organizational privacy and records retention policies.
- Personnel files are updated within 5 business days of any status change (clearance renewal, certification update, access authorization change).
- Annual personnel security review is conducted to confirm continued qualification for assigned role and access level.

## Description — Integration Notes

Personnel security qualification management is a foundational control for confidentiality, integrity, and availability assurance. It bridges organizational governance (ID21, ID06) with project-level role management (SK-D11-011, SK-D11-012, SK-D11-013). Personnel assignments to security-sensitive roles must be verified before granting access to classified systems or information. The skill requires close coordination with human resources for background verification and legal/compliance for clearance and privacy compliance. Changes to personnel security status (clearance expiration, certification loss, access violation) must trigger immediate escalation through change control (SK-D10-002, SK-D10-003) and risk re-assessment where impacts on security design exist.
