## Description

Develop Site Acceptance Testing (SAT) procedures adapted from FAT procedures (SK-D08-001) for field conditions, verifying that the system operates correctly in the actual site environment with site-specific configurations, environmental conditions, and integration with existing site infrastructure. SAT extends factory testing by validating system behavior in operational context: site network topology, site environmental conditions (temperature, humidity, EMI), integration with site IT/OT systems, and site-specific data sources and loads. The distinction between FAT, SAT, and SIT (per BOUNDARY-007) is critical: FAT validates the system as a standalone unit in a controlled factory environment; SAT validates the system functioning correctly at the site with site-specific configurations and environmental factors; SIT (System Integration Testing) validates the system integrated with other enterprise systems (typically handled separately). SAT procedures adapt FAT test cases to the site environment, add site-specific test scenarios, and establish acceptance criteria aligned with site operational requirements.

## Inputs

- FAT procedures and test cases (from SK-D08-001 ⏳: Factory Acceptance Test Development)
- Site environment specification: ambient temperature, humidity, EMI/RFI levels, altitude, corrosive atmosphere (if applicable)
- Site network topology and data flow diagrams (from SK-D02-004 ⏳)
- Site IT/OT integration requirements and existing site systems (SCADA, ERP, MES, historian)
- Site-specific data sources and loads: weather data, market data, sensor inputs
- Security FAT test cases (from SK-D08-002) adapted for site-specific authentication and network connectivity
- Site operational procedures and daily workflow (from site operations team)
- Site safety and access restrictions (hazardous areas, lockout/tagout procedures)

## Outputs

- SAT Procedure Manual: overview of SAT scope, site environment validation requirements, site-specific test scenarios, site acceptance criteria, and test execution timeline
- SAT Test Case Adaptation Document: list of FAT test cases adapted for site conditions, with site-specific modifications and rationale
- Environmental Testing Procedures: validation of system operation under site-specific environmental conditions (temperature, humidity, EMI, altitude), including monitoring and tolerance limits
- Site Integration Test Procedures: verification of connectivity with site IT/OT systems, data flow validation, network performance under site conditions, integration with existing site systems
- Site-Specific Security SAT Procedures: verification of access control using site-specific accounts and network credentials, security alarm testing using site network configuration, audit log testing with site log aggregation systems
- Site Data Source and Load Testing Procedures: validation with real site data sources (weather feeds, grid operator interfaces), load testing with representative site operational loads
- Site Operational Scenario Testing: execution of typical operational workflows using site procedures, shift handover procedures, emergency response procedures (if applicable)
- SAT Acceptance Criteria Specification: pass/fail criteria for each test scenario, success metrics aligned with site operational requirements
- SAT Report Template: structure for documenting test results, findings, remediation items, and sign-off by site operations and project stakeholders

## Tools

- Site environmental monitoring equipment: thermometer, hygrometer, electromagnetic field (EMF) meter (if high EMI/RFI environment)
- Site network connectivity tools: ping, traceroute, network analyzer (Wireshark) for network performance validation under site conditions
- System monitoring and logging tools native to the site environment (site historian, SCADA historian, syslog aggregation)
- Test case execution and documentation tools (TestRail, Azure Test Plans, or structured markdown)
- Site IT/OT system interfaces: API clients, database query tools, messaging protocol clients (for integration testing)
- Site facilities and safety equipment per site requirements (hard hats, safety glasses, access badges)
- Excel / structured markdown for SAT procedure documentation and results tracking

- IEC 62443 series (all parts): foundational IEC 62443 lifecycle framework establishing FAT/SAT/SIT distinction
- IEC 61508 Part 7: Guidelines for functional safety — system acceptance in operational environment
- NIST SP 800-82 Rev. 3: Guide to OT Security — guidance on field validation and site-specific security testing
- IEC 61010-1: Safety requirements for electrical equipment for measurement, control, and laboratory use — safety considerations during site testing
- Site-specific standards and procedures: site safety manual, environmental specifications, operational procedures

## IEC 62443 Lifecycle Stages

- R3 (Implementation — SAT procedure development and execution)
- R4 (Operations — initial operational verification before full system handover)

## Roles

- STC (Security Technical Consultant) — primary: develops SAT procedures from FAT, ensures security validation, coordinates site testing
- OPS (Operations Engineer) — supports SAT execution, validates system behavior under operational conditions, provides operational context
- QA (Quality Assurance Engineer) — executes test procedures, documents results, manages test data
- SYS (System Architect) — provides system context for site-specific test scenario design
- Site Operations Lead / Site Facility Manager — provides site environment specifications, coordinates site access, validates operational acceptance

## Dependencies

### Hard Dependencies (Required Before Execution)

- SK-D08-001 ⏳: Factory Acceptance Test Development — FAT procedures are the foundation for SAT adaptation
- Site environment specification and site network topology — essential for designing site-specific test scenarios
- Site operations team access and coordination — SAT execution requires site access, cooperation, and operational context

### Soft Dependencies (Enhances Effectiveness)

- SK-D08-002: Security FAT Test Case Design — security-specific SAT procedures are adapted from security FAT test cases
- SK-D02-004 ⏳: Data Flow Diagram Development — site-specific data flow diagrams inform site integration testing
- SK-D08-001 ⏳: FAT Execution and Documentation — FAT findings may require SAT procedure adjustments

## Acceptance Criteria

1. SAT procedure manual covers all FAT test cases with documented site-specific adaptations and rationale
2. Environmental testing procedures validate system operation under site-specific temperature, humidity, EMI/RFI, and altitude conditions with documented tolerance limits
3. Site integration test procedures cover all site IT/OT system integrations with explicit validation of data flow, connectivity, and network performance
4. Security SAT procedures validate access control with site-specific accounts and credentials, security alarms with site network configuration, and audit logging with site log aggregation
5. Site-specific data source and load testing procedures exercise the system with real site data and representative operational loads
6. Operational scenario testing covers typical operational workflows and emergency procedures (if applicable)
7. Acceptance criteria are explicitly defined per test scenario; success metrics are aligned with site operational requirements and documented with site operations sign-off
8. SAT procedures have been reviewed and approved by STC, site operations lead, and project manager; all site safety and access requirements are documented

## Estimated Effort

| Level | Effort | Notes |
|-------|--------|-------|
| Junior (< 2 yr) | 6–12 person-days | Assumes adaptation of ~30–40 FAT test cases, moderate site complexity, straightforward environmental conditions; includes SAT procedure development, site coordination, and documentation |
| Senior (5+ yr) | 3–6 person-days | Same scope; senior can leverage prior SAT experience and reusable SAT procedure templates |

Notes: Greenfield sites with simpler environments may require less effort. Brownfield sites with complex existing IT/OT infrastructure requiring extensive integration testing may require 1.5–2× effort. Remote or hazardous sites with access restrictions may add 2–3 days for coordination. Multi-site projects scale per site with some template reuse.
