## Description

Author the technical proposal document for bid/tender response, integrating the outputs of requirements analysis, feasibility assessment, security classification, concept architecture, and CBOM into a cohesive, customer-facing proposal that describes the proposed solution, delivery approach, and technical qualifications. The technical proposal is the primary customer-facing deliverable of the Pre-Gate 0 phase and must present the solution in terms the customer/employer can evaluate. Under GOV-SD, the technical proposal content is advisory (non-binding) per the Pre-Gate 0 boundary. This skill covers proposal authoring and document assembly; the individual technical analyses feeding the proposal fall under their respective skills (SK-D14-001 through SK-D14-014), and the commercial quotation that accompanies the proposal falls under SK-D14-005.

## Inputs

- Requirements specification (from SK-D14-001) — what the customer needs
- Tender security requirements (from SK-D14-010) — security scope and SL-T proposals
- Feasibility assessment (from SK-D14-003) — technical approach and risk summary
- Preliminary security classification (from SK-D14-014) — security ambition level
- CBOM (from SK-D14-005) — solution cost baseline (referenced, not embedded in technical proposal)
- Concept Zone/Conduit architecture (from SK-D14-013 ⏳) — architectural concept for visual presentation
- ICP company qualifications: certifications, relevant project experience, team qualifications
- Customer evaluation criteria (from tender documents): what the customer weights in evaluation

## Outputs

- Technical Proposal Document:
  - Executive summary: solution overview in customer-accessible language
  - Understanding of requirements: demonstration that ICP understands the customer's needs and environment
  - Proposed solution description: system architecture concept, security approach, technology selections, integration strategy
  - Delivery approach: lifecycle methodology (IEC 62443 framework), phase descriptions, milestone schedule
  - Security approach: IEC 62443 alignment, SL-T proposals, security deliverable summary
  - Team qualifications: relevant certifications (IEC 62443, CISSP), project experience, team composition
  - Risk management approach: identified risks and mitigation strategies
  - Assumptions and exclusions: clear boundary of what is included/excluded
  - Compliance matrix: point-by-point response to customer evaluation criteria or tender requirements
- Proposal Summary (2–3 pages): condensed version for executive review
- Compliance Matrix: tabular response to each tender requirement with proposal section reference

## Automation Potential

**Level:** Partial

Proposal template generation and section boilerplate can be automated. Compliance matrix generation from requirement-to-section mapping can be automated. Company qualification sections can be reused across proposals. However, solution description requires tailored authoring for each customer context, executive summary requires strategic positioning, and compliance matrix point-by-point responses require interpretive writing.

## Acceptance Criteria

1. Technical proposal addresses all customer evaluation criteria: compliance matrix covers 100% of tender evaluation points
2. Proposed solution described in sufficient technical depth for customer evaluation — architecture concept diagram included, technology selections stated
3. Security approach section includes IEC 62443 framework reference, SL-T proposals, and security deliverable inventory
4. Team qualifications section includes relevant certifications and project experience — verifiable credentials
5. Assumptions and exclusions clearly stated — no ambiguous scope boundaries in the proposal
6. Proposal marked as Pre-Gate 0 advisory per GOV-SD — no language implying firm design commitment

## Estimated Effort

| Level | Effort | Notes |
|-------|--------|-------|
| Junior (< 2 yr) | 8–12 person-days | Assumes medium-complexity tender; includes section authoring, input integration, diagrams, compliance matrix, and review cycles |
| Senior (5+ yr) | 5–8 person-days | Same scope; senior leverages proposal templates and efficient narrative writing |

Notes: Effort depends on customer tender complexity and evaluation criteria detail. Large government tenders with extensive compliance matrices may require 2× effort. International tenders requiring translation add overhead. Proposal writing is often the most time-pressured Pre-Gate activity due to tender deadlines. Quality of upstream inputs (requirements, feasibility, CBOM) significantly affects proposal writing efficiency.
