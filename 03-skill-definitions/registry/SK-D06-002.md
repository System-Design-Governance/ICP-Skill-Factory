## Description

Develop detailed wiring diagrams showing all electrical connections between components within control panels and between panels and field devices. Wiring diagrams are the definitive record of physical point-to-point electrical connections and are essential for panel assembly, field installation, commissioning, and maintenance. This skill encompasses wire numbering conventions, terminal references, cable schedules, connection verification marks, and adherence to IEC 61346 (Structured Principles and Reference Designations for Electric Devices and Systems) and IEC 60617 (Graphical symbols for use in electrical and electronics engineering). Diagrams are typically produced using AutoCAD Electrical or EPLAN Electric P8 to ensure consistency, automatic bill-of-materials generation, and cross-referencing with terminal block schedules (SK-D06-003) and fabrication drawings (SK-D06-004).

## Inputs

- Functional electrical schematic diagrams (from SK-D04-001 ⏳: Functional Electrical Schematic Design; or delivered as part of detailed design package)
- Equipment data sheets and datasheets (relay modules, drives, power supplies, I/O modules)
- Terminal block schedule (from SK-D06-003: Terminal Block Schedule Development; or developed iteratively)
- Wire sizing calculation results (from SK-D06-005: Wire Sizing Calculation)
- Control panel layout and fabrication drawings (from SK-D06-004: Fabrication Drawing Production)
- Cable schedule with cable IDs, types, and routing paths (from SK-D06-002 or SK-D02-003 ⏳: Interface Control Document)
- Customer wiring and labeling standards documentation

## Outputs

- Complete wiring diagram set in PDF and CAD formats (.dwg for AutoCAD Electrical or .eplan for EPLAN)
- Wire number list: wire ID, source terminal, destination terminal, wire gauge, color code, cable reference
- Cable schedule reference matrix linking wiring diagram wire numbers to physical cable labels
- Connection verification report: checklist of all connections verified against schematic and component datasheets
- Revision-controlled diagram file with tracking of engineering changes per SK-D09-005

## Tools

- AutoCAD Electrical (primary; supports automated wire numbering, terminal cross-referencing, and BoM generation)
- EPLAN Electric P8 (alternative; integrates with EPLAN's harness planning module for cable management)
- Excel / structured markdown (wire number lists, cable schedules)

## Standards

- IEC 60617: Graphical symbols for use in electrical and electronics engineering — symbol definitions and wiring diagram conventions
- IEC 61346: Structured Principles and Reference Designations for Electric Devices and Systems — wire and component labeling conventions
- IEC 61810-1: Electromechanical elementary relays — device reference designations
- IEC 62061 / IEC 61131-2: Functional safety and control system standards (if safety-related)
- Customer/regional electrical codes (NEC, IEC 60364 in Europe; local distribution utility standards for field-mounted devices)

## IEC 62443 Lifecycle Stages

- R2 (Detailed Design) — final wiring diagram is a key deliverable of detailed design phase, enabling fabrication and field installation

## Roles

- DES (Design Engineer) — primary: develops wiring diagrams, maintains CAD models, coordinates with panel fabrication and installation teams
- SYS (System Architect) — provides system and subsystem topology; reviews wiring diagrams for compliance with system architecture
- Manufacturing/Panel Integration Engineer — reviews diagrams for fabrication feasibility and suggests layout optimizations

## Dependencies

### Hard Dependencies (Required Before Execution)

- SK-D04-001 ⏳: Functional Electrical Schematic Design — wiring diagram is derived from the detailed functional schematic
- SK-D06-005: Wire Sizing Calculation — wire gauges and cable types must be determined before finalizing wiring diagrams
- SK-D06-003: Terminal Block Schedule Development — terminal block definitions are cross-referenced in wiring diagrams

### Soft Dependencies (Enhances Effectiveness)

- SK-D06-004: Fabrication Drawing Production — fabrication drawings show panel layout, which informs physical routing of wires and terminal block placement
- SK-D02-003 ⏳: Interface Control Document — defines inter-panel communication cables that must be shown in field wiring portions of diagram
- SK-D09-005: Engineering Change Control — wiring diagrams must be revision-controlled and updated for all approved engineering changes

## Acceptance Criteria

1. Every wire in the functional schematic is explicitly numbered in the wiring diagram (no ambiguous or unlabeled connections)
2. Every wire number is cross-referenced to source terminal, destination terminal, and cable reference (if applicable)
3. Wire gauges and colors match the results of SK-D06-005 (Wire Sizing Calculation)
4. All terminal block references point to valid entries in SK-D06-003 (Terminal Block Schedule)
5. Diagram is drawn to IEC 60617 symbol standards and IEC 61346 reference designation conventions
6. Cable schedules are complete: every inter-panel cable and field device cable is listed with ID, type, length, and color code
7. Connection verification report documents 100% review of all connections against source schematic and datasheets
8. Diagram has been reviewed and approved by DES lead and reviewed by the System Architect for consistency with system-level architecture

## Estimated Effort

| Level | Effort | Notes |
|-------|--------|-------|
| Junior (< 2 yr) | 5–8 person-days | Single panel or modular subsystem; includes symbol drawing, wire numbering, and terminal cross-referencing |
| Senior (5+ yr) | 2–3 person-days | Same scope; senior can leverage templates, macro automation, and cross-reference functionality in CAD tools |

Notes: Effort scales with diagram complexity (number of I/O points, inter-panel connections) and number of revisions driven by design iterations. Multi-panel systems may require 1.5–2× effort. EPLAN-based design may be faster if prior EPLAN projects exist in the organization.
