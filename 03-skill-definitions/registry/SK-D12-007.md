## Description

This skill encompasses the development of a comprehensive master Data Dictionary that serves as the authoritative reference for all data elements across OT and IT systems in the project scope. The Data Dictionary documents for each element: field name (in English and Chinese if applicable), data type (integer, float, string, boolean, enumeration), physical unit (MW, °C, psi, etc.), value range and valid values, data source system and update frequency, access classification per SK-D01-033 (Data Classification Policy), data owner, and business/operational meaning. The Data Dictionary enables consistent data interpretation across SK-D07-005 (Cross-System Data Model Alignment), SK-D12-001 (Data Acquisition Architecture), SK-D12-003 (Time-Series Database Selection), and SK-D12-006 (Analytics Dashboard Design). It serves as the single source of truth for data governance, supporting data quality monitoring, analytics development, and system integration.

## Inputs

- Data Acquisition Architecture (from SK-D12-001: Data Acquisition Architecture) — identifies data sources and elements to be captured
- Cross-System Data Model Alignment (from SK-D07-005: Cross-System Data Model Alignment) — mapping of data elements across OT and IT systems
- Data Classification Policy (from SK-D01-033: Data Classification Policy) — classification levels for access control
- System Design Documentation (from SK-D01-001 through D07) — technical specifications and design context for data elements
- Operational Requirements and KPIs: business/operational meaning and units for data elements
- Hardware and Software Specifications: device parameters, sensor specifications, application field definitions
- Standards and Naming Conventions: organizational data naming conventions and metadata standards

## Outputs

- Master Data Dictionary: comprehensive document organized by system/domain, listing all data elements with metadata (name, type, unit, range, source, classification, owner, definition)
- Data Element Metadata Catalog: structured database or spreadsheet with searchable index of all data elements and their properties
- Data Naming Conventions and Standards: guidelines for data element naming, abbreviations, units, and metadata documentation
- Data Classification Mapping: alignment of data elements with sensitivity classifications from SK-D01-033
- Data Ownership Registry: assignment of data owners/stewards responsible for data quality and governance
- Data Lineage Diagram: visual representation showing how data flows from source systems through transformations to destination systems
- Glossary and Abbreviations: definitions of technical terms and system-specific abbreviations used in data elements

## IEC 62443 Lifecycle Stages

- R2 (Detailed Design) — Data Dictionary is developed to document all data elements and metadata
- R3 (Implementation) — Data Dictionary is used to configure databases, APIs, and data pipelines
- R4 (Commissioning/Startup) — Data Dictionary is verified against actual deployed systems; discrepancies are resolved
- R5 (Operations) — Data Dictionary is maintained and updated as systems evolve; new data elements are added per governance process

## Acceptance Criteria

1. Data Dictionary includes 100% of data elements from all identified OT and IT systems; coverage is verified against Data Acquisition Architecture
2. For each data element: name (English and Chinese if applicable), data type, physical unit, value range (min/max or enumeration), source system, and update frequency are documented
3. All data elements are assigned to data owners/stewards; ownership is documented with contact information and responsibilities
4. Data classification per SK-D01-033 is assigned to each element; sensitive data is marked appropriately
5. Data Dictionary is organized for usability: searchable index, grouped by system/domain, with cross-references and related elements documented
6. Data Dictionary has been reviewed and approved by System Architect, Data Governance Officer, and subject matter experts (control engineers, operators); accuracy of data types and units is verified
