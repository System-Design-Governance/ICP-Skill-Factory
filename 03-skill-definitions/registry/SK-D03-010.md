## Description

Build and maintain power system simulation models in ETAP and/or PSS/E software platforms, including comprehensive network topology representation, component parameterization from equipment datasheets and field measurements, and validation of model behavior against historical field data and commissioning test results. This is the foundational modeling skill that enables all downstream power system analysis skills: SK-D03-001 (Power Flow), SK-D03-002 (Short Circuit), SK-D03-003 (Voltage Stability), SK-D03-004 (Reactive Power Compensation), SK-D03-009 (Transient Stability), and SK-D03-008 (Harmonic Analysis). Model accuracy and maintainability directly impact the reliability of security architecture decisions driven by these analyses.

## Inputs

- Network one-line diagram (electrical topology):
  - Substations, buses, transmission/distribution feeders
  - Generation and storage facilities
  - Transformers, reactors, capacitors, harmonic filters
  - Transmission lines and distribution cables (impedance data)
  - Loads and demand profiles
- Equipment datasheets and specifications:
  - Synchronous generators: rated power, voltage, frequency, subtransient/transient reactances, short-circuit ratio, inertia constant (H)
  - Transformers: ratings, impedance (Z%), tap ratios, winding connections, cooling ratings
  - Motors and adjustable frequency drives: rated power, starting characteristics, control modes
  - Transmission lines/cables: conductor type, span length, frequency-dependent resistance and reactance
  - Circuit breakers: rated voltage, current, interrupting capacity
  - Relays and protection devices: operation curves, settings
  - FACTS/inverter-based resources: control models, capability curves
- Field measurement data:
  - Power flow snapshots (MW, MVAR, voltage, frequency) at representative operating points
  - Three-phase voltage and current measurements at critical buses
  - Frequency response test data (generator inertia verification)
  - Motor starting tests (voltage drop during start)
  - Harmonic spectrum measurements (if commissioning includes power quality survey)
- Historical disturbance records (oscillograph or relay logs) from system faults or events
- Connection and interface specifications (if integrating new DER or interconnection)
- System diagrams and protection settings documentation

## Outputs

- ETAP and/or PSS/E Power System Model Files:
  - Complete network model with all buses, branches, and components
  - Validated against baseline power flow and short-circuit data
  - Version control with change log (model evolution over project phases)
  - Model database backup and disaster recovery procedures
- Model Documentation Package:
  - Network diagram (one-line) with component identification numbers cross-referenced to model
  - Component parameters table: generators, transformers, lines, loads, protective devices with source data
  - Model assumptions and simplifications (e.g., lumped loads, negligible impedance branches)
  - Bus naming convention and coordinate reference system
  - Frequency base and power base (MVA) used in model
  - Control model descriptions (AVR, governor, PSS tuning)
  - Line/cable impedance derivation (frequency, temperature, bundling corrections)
- Model Validation Report:
  - Baseline power flow comparison (model load flow vs. measured or typical operating point):
    - Bus voltage magnitude and angle
    - Line/transformer power flows
    - Generator power and reactive output
  - Short-circuit validation: model fault current vs. field test or nameplate data
  - Transient response validation (where historical disturbance records available): generator frequency response, oscillation damping
  - Sensitivity analysis: impact of model parameter uncertainties on load flow and short-circuit results
- Model Training and Maintenance Documentation:
  - Standard operating procedures for model updates (adding new equipment, changing protection settings)
  - Data quality and source documentation standards
  - Archiving and version control procedures
  - Annual model audit checklist

## Tools

- ETAP (Electrical Transient Analyzer Program): Power Systems Analysis module
- PSS/E (Power Systems Simulator for Engineering): Version 34.x or later
- MATLAB/Simulink (supplementary dynamic model development if custom control models required)
- Python (model scripting, batch case generation, post-processing)
- Excel (component parameter tables, measurement data organization)
- Git or Subversion (version control for model files)
- Power system measurement instruments: power quality analyzer, frequency measurement device, phase angle recorder

## Standards

- IEEE Std 399: IEEE Recommended Practice for Industrial and Commercial Power Systems Analysis — modeling methodology and validation
- IEEE Std 1015: IEEE Guide for Induction Machinery Maintenance Testing and Failure Analysis — motor parameter derivation
- IEC 60071-1: Insulation Coordination — voltage classifications and equipment ratings
- IEC 61000-2-2: Electromagnetic compatibility — network impedance and harmonics background
- ETAP and PSS/E User Manuals: tool-specific modeling conventions and best practices
- NERC Reliability Standards MOD (Modeling): requirements for power system model accuracy and maintainability (NERC MOD-032, MOD-033)
- IEC 62443-3-2: Security Risk Assessment — model accuracy as a prerequisite for valid security analysis

## IEC 62443 Lifecycle Stages

- R1 (Security Requirements) — preliminary model for risk assessment and threat analysis
- R2 (Detailed Design) — refined model with validated parameters supporting detailed design decisions
- R3 (Implementation Specification) — model updated with field-measured equipment parameters
- R4 (Implementation and Integration Assurance) — model validation against commissioning test results; ongoing maintenance

## Roles

- SYS (System Architect) — primary: oversees model development strategy, validates model scope and assumptions
- ENG (Design Engineer) — builds and maintains the model in ETAP/PSS/E, performs parameter research
- PRAC (Power System Practitioner) — validates model against field experience and historical data, identifies parameter refinements
- OPS (Operations Engineer) — provides operating point data and disturbance records for model validation

## Dependencies

### Hard Dependencies (Required Before Execution)

- Network topology diagram and equipment inventory: cannot build model without knowing what exists
- Generator and transformer datasheets: essential for synchronous machine and transformer impedance parameters

### Soft Dependencies (Enhances Effectiveness)

- Historical power flow or load flow study results: accelerates model baseline calibration
- Field measurements or commissioning test reports: validates model accuracy and identifies parameter refinements
- Relay coordination study: protection settings inform dynamic model control parameters (relay models, control time constants)
- SCADA/EMS historian data: provides representative operating points across all seasons and load conditions

## Acceptance Criteria

1. Network topology 100% complete: all buses, branches, and equipment per approved one-line diagram
2. Component parameters sourced from datasheets, field tests, or prior validated studies with documented traceability
3. Model power flow converges at minimum 3 representative operating points (peak, minimum, seasonal variations)
4. Validated power flow results match measured or industry-standard data within 5% for bus voltage, 3% for generator power
5. Short-circuit model validation: calculated fault current within 10% of field test results or per ANSI/IEEE guidelines
6. If available, transient response model validated against historical disturbance records: generator frequency response and oscillation damping agreement within 10%
7. All component parameter assumptions documented and signed-off by SYS and PRAC
8. Model file version control established with change log and configuration management procedures
9. Model documentation complete: one-line diagram, parameter tables, assumptions, validation report

## Estimated Effort

| Level | Effort | Notes |
|-------|--------|-------|
| Junior (< 2 yr) | 15–25 person-days | Assumes medium-complexity system (10–20 generators, 30–50 buses), greenfield project; includes topology entry, parameter research, validation |
| Senior (5+ yr) | 8–12 person-days | Same scope; senior leverages template models and parameter databases; faster research and validation |

Notes: Brownfield projects with legacy equipment documentation or missing data may require 1.5–2× effort for parameter estimation and field verification. Microgrids or systems with many DER/inverters add 5–10 days for control model development. Integration of new DER or protection system changes typically requires 2–5 days of model maintenance per project.
