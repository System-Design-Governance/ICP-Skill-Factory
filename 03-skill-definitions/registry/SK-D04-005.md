## Description

Analyze fault recording data from digital fault recorders (DFRs) and protection relay event logs to diagnose power system fault events, determine root causes, and verify protection system operation. This skill uses waveform analysis, sequence-of-events reconstruction, and protection operation verification to establish detailed understanding of how power system faults developed and how protection systems responded. Fault recording analysis is essential for power system reliability engineering, protection setting refinement, and incident investigation in electrical generation, transmission, and distribution systems. Analysis results feed protection setting refinement (SK-D04-001: Protection Setting Validation; SK-D04-002: Protection Setting Optimization) and incident investigation (SK-D01-017: Incident Investigation) with technical findings on protection system effectiveness and operational challenges. This skill implements industry standard practice (PRAC) for power system fault analysis and complements SK-D08-007 (Performance Baseline) for normal operational baselines against which fault conditions are evaluated.

## Description — Objectives

- Extract and interpret fault recording data from digital fault recorders and protection relay event logs
- Perform waveform analysis to characterize fault type, location, and severity
- Reconstruct chronological sequence of events showing fault development and protection operation
- Verify protection system operation and response timing against protection setting design
- Identify protection system deficiencies or unexpected behavior
- Determine root cause of fault event and contributing factors (equipment failure, maintenance issues, operational factors)
- Produce technical analysis report with findings and recommendations for protection refinement
- Support security incident investigation by distinguishing operational faults from intentional malicious activities

## Description — Scope

**In Scope:** Digital fault recorder data extraction and interpretation; waveform analysis (voltage, current, frequency); sequence-of-events reconstruction; protection relay operation verification; transient overvoltage and harmonic analysis (where applicable); root cause determination; protection operation anomaly identification; technical analysis report generation.

**Out of Scope:** Real-time monitoring and control system operations (separate operational function); power system steady-state analysis and load flow (separate engineering discipline); remediation implementation (assigned to engineering teams); protection scheme redesign (scope of SK-D04-001/002).

## Inputs

- Digital fault recorder data files from deployed DFRs (high-resolution waveform samples from fault event)
- Protection relay event logs and disturbance records (timestamp, protection function, trip signal, measured values)
- Protection setting documentation (relay curves, time delays, pickup thresholds) from SK-D04-001 or SK-D04-002
- System architecture and single-line diagram showing equipment arrangement, protection zones, and relay locations
- Operational baseline data (normal voltage, current, frequency profiles) from SK-D08-007 (Performance Baseline)
- Historical fault event database or prior analysis reports for similar fault types (for comparison)
- Equipment specifications (transformer ratings, line parameters, generator capability) for system modeling
- Organizational incident investigation procedures (SK-D01-016, SK-D01-017) if fault is triggered by suspected malicious activity

## Description — Key Concepts

- **Digital Fault Recorder (DFR):** High-resolution instrument that continuously samples voltage and current waveforms and stores data during power system fault events; provides time-stamped waveform data for offline analysis
- **Waveform Analysis:** Examination of voltage and current waveforms to determine fault type (phase-to-phase, phase-to-ground, three-phase), location, and severity
- **Sequence of Events (SOE):** Chronological record of protection relay operations and other critical events during a fault with millisecond or better time resolution
- **Protection Operation Verification:** Confirmation that protection relay correctly detected fault, calculated trip decision according to setting, and issued trip command to circuit breaker at appropriate time
- **Transient:** Temporary deviation from steady-state conditions occurring during power system disturbances
- **Relay Curve:** Characteristic relationship between current (or other measured quantity) and operating time that defines when relay issues trip command
- **Root Cause Analysis:** Determination of fundamental reason why fault occurred and why protection response succeeded or failed

## Description — Methodology

Fault recording analysis for power system protection follows a structured five-phase technical process:

**Phase 1: Fault Recording Data Collection and Validation**
- Retrieve digital fault recorder data files from affected location(s):
  - Three-phase voltage waveforms (Va, Vb, Vc) sampled at minimum 1 kHz (typically 4–16 kHz)
  - Three-phase current waveforms (Ia, Ib, Ic) sampled at same rate as voltage
  - Disturbance start time and duration (pre-fault and post-fault windows typically 0.5–1 second)
- Retrieve protection relay event logs:
  - Sequence-of-events timestamps for protection function activations
  - Measured voltage and current values at time of trip decision
  - Protection function operated (overcurrent, distance, differential, etc.)
  - Trip signal timestamp and circuit breaker opening confirmation
- Validate data quality:
  - Confirm waveform sampling rate is adequate for fault frequency analysis (minimum Nyquist sampling rate for maximum expected harmonic content)
  - Verify time synchronization between DFR and relay event logs (typically GPS/PTP synchronized)
  - Identify any data gaps or corruption
  - Confirm event window includes pre-fault steady state and post-fault clearing

**Phase 2: Waveform Analysis and Fault Characterization**
- Perform waveform analysis to characterize fault:
  - **Fault Type Identification:** Examine phase voltage drops and current magnitudes to determine fault type:
    - Single-phase-to-ground (SLG): One phase voltage near zero, other phases elevated
    - Phase-to-phase (LL): Two phase voltages depressed, third phase normal
    - Three-phase (3LG): All three phase voltages collapsed to near zero
  - **Fault Magnitude:** Calculate RMS current in fault phase(s) to assess severity (higher current = more severe fault closer to protection location)
  - **Fault Location:** Use impedance or traveling wave methods to estimate distance to fault from relay location
    - Impedance method: Calculate impedance seen by relay (V/I ratio) and compare to known line impedance
    - Traveling wave method: Analyze waveform transient arrival time at multiple relay locations to triangulate fault position
  - **Transient Characteristics:** Analyze transient overvoltages, inrush currents, and harmonic content to assess equipment stress
- Document fault characteristics with supporting waveform plots and calculated values

**Phase 3: Sequence-of-Events Reconstruction**
- Build chronological timeline of fault event:
  - t0 = Fault inception (when first waveform deviation occurs)
  - t1 = Relay function 1 detects fault (pickup time, measured current/voltage value when pickup occurred)
  - t2 = Relay function 2 operates (if multiple protection stages in use)
  - ...
  - tn = Circuit breaker receives trip command (relay output to breaker coil)
  - tn+1 = Circuit breaker opens (confirmed by zero current or arcing contacts data)
  - tn+2 = Fault clears (when voltage and current return to normal)
- Document measured times:
  - Detection time (from fault inception to relay pickup)
  - Decision time (from pickup to trip command issuance)
  - Operating time (from trip command to circuit breaker opening)
  - Clearing time (from trip command to fault current zero crossing)
- Identify any abnormal delays or unexpected events (e.g., multiple relays operating in sequence, re-strikes, equipment failures)

**Phase 4: Protection Operation Verification**
- Verify each protection relay function operated correctly:
  - **Overcurrent Protection:**
    - Measured current exceeded pickup setting: Yes/No (expected: Yes)
    - Relay time dial and curve setting applied correctly: Yes/No
    - Time to trip matched characteristic curve: Yes/No
    - Trip command issued at correct instant: Yes/No
  - **Distance Protection (impedance relay):**
    - Calculated impedance entered zone 1 at correct time: Yes/No
    - Zone time delay applied (if configured): Yes/No
    - Trip decision logic executed correctly: Yes/No
  - **Differential Protection:**
    - Differential current magnitude exceeded threshold: Yes/No
    - Harmonic restraint logic functioned correctly: Yes/No
    - Trip issued without false trips on inrush: Yes/No
  - **Special Protection Schemes (if applicable):**
    - Fast trip logic, reclosing logic, load shedding logic all operated as designed
- Compare measured protection response against setting design:
  - Does measured pickup current match setting? (within 5% tolerance acceptable)
  - Does measured operating time match relay curve? (within time dial error tolerance acceptable)
  - Did protection operate in correct sequence? (e.g., primary zone 1 fast trip, backup after time delay)
  - Is there evidence of sympathetic trips, cascading failures, or unexpected coordination issues?

**Phase 5: Root Cause Determination and Recommendations**
- Analyze root cause of fault event:
  - **Equipment Failure:** Insulation breakdown, component deterioration, mechanical failure of equipment
    - Evidence: Sustained high current in fault phase, rapid voltage collapse, equipment damage indicators
  - **Environmental Factors:** Lightning strike, severe weather (wind, ice), vegetation contact, animal contact
    - Evidence: Single-phase fault from clear sky, correlation with weather events
  - **Maintenance Deficiency:** Inadequate inspection/maintenance creating hazardous condition
    - Evidence: Known equipment age, history of similar faults, deferred maintenance
  - **Operational Factors:** Human error, procedure violation, overload condition
    - Evidence: Unusual operational mode, confirmation from control center logs
  - **Security Incident:** Suspected malicious activity triggering fault (rare but investigate if circumstances suspicious)
    - Evidence: Unusual timing, correlation with documented security incidents, anomalous fault characteristics
- Assess protection system response:
  - Did protection operate as designed? (Yes/No with evidence)
  - Were any protection deficiencies revealed? (e.g., slower-than-expected clearing time, unexpected sympathetic trips)
  - Are protection setting adjustments recommended? (coordinate with SK-D04-001/002)
  - Is improved monitoring or redundancy recommended? (coordinate with SK-D08-007)
- Document recommendations:
  - Protection refinement (SK-D04-001: Setting Validation; SK-D04-002: Setting Optimization)
  - Equipment inspection or replacement (assign to maintenance engineering)
  - Operational procedure improvement (coordinate with operations team)
  - Enhanced monitoring (if fault recurrence risk is high)

## Description — Activities

1. Establish fault recording analysis readiness:
   - Identify and inventory all digital fault recorders deployed in power system
   - Establish data retrieval procedures from DFRs (download data, file format conversion, quality check)
   - Obtain protection relay event log access from SCADA/protective relay systems
   - Acquire fault analysis software tools (specialized software or MATLAB/Python for waveform analysis)
   - Train engineering team on DFR data interpretation and fault analysis methodology

2. Upon fault event occurrence:
   - Retrieve DFR data files from affected location(s)
   - Retrieve protection relay event logs and circuit breaker status
   - Validate data quality and time synchronization
   - Perform preliminary assessment to determine if fault is routine (weather-related, equipment failure) or suspicious (potential security incident)

3. Conduct fault analysis:
   - Perform waveform analysis and fault characterization per Phase 2 procedures
   - Reconstruct sequence-of-events per Phase 3 procedures
   - Verify protection operation per Phase 4 procedures
   - Determine root cause per Phase 5 procedures

4. Generate analysis report and recommendations:
   - Prepare formal Fault Analysis Report with waveform plots, calculated values, sequence-of-events timeline, and findings
   - Document protection operation assessment and any deficiencies identified
   - Provide recommendations for protection refinement (to SK-D04-001/002), equipment maintenance, or operational improvements
   - If security incident suspected, coordinate with SK-D01-016/017 incident response and investigation

5. Continuous improvement:
   - Share fault analysis findings with protection engineering team for incorporation into setting validation/optimization
   - Maintain database of fault events for trend analysis and pattern recognition
   - Use analysis findings to enhance SK-D08-007 (Performance Baseline) understanding of normal vs. abnormal conditions

## Outputs

- **Digital Fault Recorder Data Archive:** Stored DFR data files from fault event with metadata (location, timestamp, triggering event)
- **Fault Analysis Report:** Executive summary, waveform analysis results, sequence-of-events timeline, protection operation verification, root cause determination, findings and recommendations
- **Waveform Analysis Plots:** Three-phase voltage and current waveforms with annotations showing fault characteristics, protection function detection points, and trip times
- **Sequence-of-Events Timeline:** Detailed chronological record with timestamps showing fault inception, relay operation, trip command, and fault clearing with measured values
- **Protection Operation Assessment:** Verification that each protection function operated correctly with supporting calculations and comparisons to setting design
- **Root Cause Analysis Report:** Technical findings on fault origin, contributing factors, and any equipment deficiencies identified
- **Recommendations Document:** Suggested actions for protection refinement (SK-D04-001/002), maintenance, operational improvements, or enhanced monitoring
- **Fault Event Database Entry:** Catalogued record of fault event with analysis results for trend analysis and future reference

## Acceptance Criteria

- Fault recording data is successfully extracted from DFRs and validated for time synchronization and data quality
- Waveform analysis characterizes fault type (SLG/LL/3LG), magnitude, and location with supporting calculations
- Sequence-of-events timeline is reconstructed with millisecond-level timing showing fault inception through clearing
- Protection relay operation is verified against setting design; any deviations are documented and explained
- Root cause of fault event is determined and documented with supporting evidence and analysis
- If protection deficiencies are identified, specific recommendations for SK-D04-001 (Setting Validation) or SK-D04-002 (Setting Optimization) are provided
- Formal analysis report is produced with waveform plots, timeline graphics, and clear findings suitable for stakeholder review
- If security incident is suspected, analysis report is coordinated with SK-D01-016/017 incident response procedures
- Fault event and analysis results are documented in permanent database for trend analysis and pattern recognition
- Recommendations are tracked and incorporated into protection engineering and maintenance planning

## Description — Practical Guidance

- **Pitfall:** Analyzing incomplete fault recording data (missing phase, inadequate pre-fault or post-fault window). **Guidance:** Establish data retrieval procedures to ensure complete 3-phase data with minimum 0.5 second pre-fault window and adequate post-fault window to capture clearing; validate DFR is configured correctly for event recording.
- **Pitfall:** Time synchronization mismatch between DFR and relay event logs, making sequence-of-events reconstruction unreliable. **Guidance:** Verify DFR and relay use common time source (GPS/PTP synchronized); if not synchronized, establish time offset correction and document in analysis report.
- **Pitfall:** Confusing waveform distortions from fault transients with equipment failures. **Guidance:** Understand typical transient characteristics for different fault types; use high-frequency waveform analysis to distinguish transient overvoltages from sustained equipment faults.
- **Pitfall:** Verifying protection operation without checking against actual setting values stored in relay. **Guidance:** Retrieve protection relay settings (not just setting procedure); confirm DFR/relay data align with settings in effect at time of fault (settings may have changed since commissioning).
- **Pitfall:** Root cause analysis based on single factor without considering multiple contributing factors. **Guidance:** Systematically evaluate equipment failure, environmental factors, maintenance history, operational context, and (if appropriate) security implications; use structured root cause analysis method (5 Whys, fault tree, etc.).
- **Pitfall:** Fault analysis findings are not communicated to protection engineering team for setting refinement. **Guidance:** Coordinate analysis findings with SK-D04-001 (Setting Validation) and SK-D04-002 (Setting Optimization) teams; provide specific recommendations for setting adjustments if protection deficiencies identified.
