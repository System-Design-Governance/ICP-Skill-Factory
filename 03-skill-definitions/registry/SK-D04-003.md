## Description

Select appropriate protection relays (overcurrent, distance, differential, impedance-based, etc.) based on system requirements, operational constraints, and fault clearing objectives. Configure relay parameters including pickup values, time delays, communication settings, and IEC 61850 GOOSE subscriptions to enable coordinated protection and real-time monitoring. This skill translates protection scheme design (from SK-D04-001) into concrete relay hardware and software configurations, ensuring compliance with IEC 62443 security requirements and operational safety standards. Configuration includes setting sensitivity thresholds, time delays (instantaneous and time-delayed), communication protocol bindings, and integration with supervisory control systems per D04/D05 requirements.

## Inputs

- Protection Scheme Specification (from SK-D04-001: Protection Scheme Design)
- System Impedance Data: source impedance, line/equipment impedance values, fault MVA data
- Load and Generation Operating Ranges: minimum/maximum load profiles, DG capacity and ramp rates
- Relay Hardware Specifications: nameplate data, time-delay curve families (IEC, US, etc.), available input/output modules
- IEC 61850 Protocol Mapping (from SK-D04-001 ⏳: IEC 61850 Modeling)
- Communication Network Architecture (from SK-D02-001: OT Network Topology Design)
- Customer Requirements and Utility Standards (pickup tolerance, coordination margins, communication availability assumptions)

## Outputs

- Relay Selection Summary: relay type, manufacturer, model, nameplate ratings, installed location
- Relay Parameter Setting Sheet: pickup values (A or V), time delays, curve characteristics, communication settings per relay
- IEC 61850 GOOSE Subscription List: data attributes subscribed, logical nodes involved, reporting frequency
- Relay-to-Relay Coordination Report: coordination curves, clearing times, margin analysis vs. upstream/downstream devices
- Relay Security Configuration: access control settings (per SK-D01-020), change management procedures (per SK-D10-003), firmware versioning
- Integration Specification: I/O wiring diagram, hardwired logic interconnections, protocol bindings

## Tools

- ETAP / PSS/E (short-circuit analysis, coordination studies, relay curve plotting)
- Relay Configuration Software: manufacturer-specific tools (SEL, Schweitzer, ABB, Siemens, GE)
- IEC 61850 MMS/GOOSE Configuration Tools (per SK-D04-001 output)
- AutoCAD / P&ID drawing tools (control wiring, installation details)
- Spreadsheet / structured markdown (parameter tables, coordination matrices)

## Standards

- IEC 60255 series: Protection relays and measuring relays — functional characteristics and settings
- IEC 61850-8-1: Communication networks and systems in substations — specific communication service mapping (GOOSE)
- IEC 62443-3-3: System Security Requirements and Security Levels — control system security requirements
- IEC 62443-4-1: Security of Product Development — secure relay firmware and configuration management
- IEEE C37.90.1: Standard for Surge Withstand Capability Tests for Protective Relays and Instrument Transformers
- Utility-specific relay setting guides (e.g., WECC, ReliabilityFirst standards)

## IEC 62443 Lifecycle Stages

- R2 (Detailed Design — relay selection and parameter configuration based on finalized protection scheme)
- R3 (Implementation — verification that installed relays match specification and are correctly configured)

## Roles

- DES (Design Engineer) — primary: selects relays, calculates and verifies settings, produces parameter sheets
- STC (Security Technical Consultant) — supports with communication protocol security and IEC 61850 GOOSE security bindings
- TST (Test Engineer) — validates settings against test cases and coordination study assumptions
- OPS (Operations Engineer) — provides operational input on time delays, load margins, and real-world setting constraints

## Dependencies

### Hard Dependencies (Required Before Execution)

- SK-D04-001 ⏳: Protection Scheme Design — defines which protection types are needed and their functional requirements
- SK-D04-001 ⏳: IEC 61850 Modeling — establishes logical nodes and GOOSE data attributes to be subscribed
- SK-D02-001 ⏳: OT Network Topology Design — network architecture affects communication availability and backup logic

### Soft Dependencies (Enhances Effectiveness)

- SK-D01-020 ⏳: Cybersecurity Access Control Design — relay access control strategy
- SK-D10-003 ⏳: System Change Management Procedure Design — relay firmware/configuration change procedures
- SK-D04-002 ⏳: SCADA Monitoring and Alarm Design — alarm strategy for relay trip events and communication failures

## Acceptance Criteria

1. All relays specified in the Protection Scheme Design (SK-D04-001) are selected with clear model numbers and nameplate ratings
2. Relay parameters (pickup, time delays, curve type) are calculated from system impedance data with documented assumptions
3. Coordination margin (min. 0.3 s) is maintained between all relay pairs in the protection hierarchy
4. All GOOSE subscriptions are mapped to IEC 61850 logical nodes with confirmed data availability from MMS servers
5. Relay security settings (password, firmware version, authentication method) comply with SK-D01-020 access control requirements
6. Parameter sheets have been reviewed and approved by the System Architect (SYS) and Operations Engineer (OPS)
7. Setting calculations are traceable to the short-circuit study and load flow analysis

## Estimated Effort

| Level | Effort | Notes |
|-------|--------|-------|
| Junior (< 2 yr) | 4–7 person-days | Assumes 5–10 relays, straightforward protection scheme, manufacturer-provided calculation tools |

Notes: Complex multi-terminal lines, DC protection, or non-standard schemes may require 1.5–2× effort. IEC 61850 GOOSE subscription configuration adds 1–2 days depending on system complexity.
