## Description

Site Integration Test (SIT) Protocol Development defines the comprehensive testing approach for verifying end-to-end system integration across multiple subsystems in the actual site environment, distinct from Factory Acceptance Test (FAT) and Subsystem Acceptance Test (SAT). SIT validates cross-system data flows, protocol interoperability, integrated security operation, and system behavior under realistic operational scenarios in the deployed facility. Unlike SAT, which tests individual subsystems in isolation, SIT focuses on interactions between SCADA, EMS, DERMS, historian, protection, and other integrated systems operating together. The SIT protocol (R3) establishes test categories, test cases, success criteria, and procedures that guide integration verification before system handover to operations.

- Test data generation procedures and representative data sets for realistic scenarios
- Test environment setup and configuration guide for site conditions
- Traceability matrix mapping test cases to system requirements and integration interfaces
- SIT test execution report and pass/fail assessment

## IEC 62443 Lifecycle Stages

- **R2 (Design Development)**: Review and refine SIT test protocol based on detailed design outputs from SK-D07-002 through SK-D07-006.
- **R3 (Implementation Verification)**: Execute SIT test protocol to verify that implemented integration meets R3 security requirements and functional specifications.
- **R4 (Commissioning)**: Prepare SIT results for handover to operations; address any identified deficiencies before production deployment.

## Roles

- **STC**: Security Technical Consultant leads SIT protocol development and oversees security-related test categories.
- **TEST**: Test Engineer designs test cases, manages test execution, and documents results.
- **OPS**: Operations Engineer validates that SIT scope and test cases cover operational scenarios and safety-critical functions.
- **ENG**: Integration Engineer supports SIT test definition with technical understanding of subsystem interfaces.

## Acceptance Criteria

1. SIT test protocol document is complete and identifies all test categories required to verify cross-system integration, including data flow verification, protocol interoperability, security operation integration, and performance under load.
2. Test cases are specified for all critical integration interfaces with clear preconditions, step-by-step procedures, expected results, and quantitative acceptance criteria.
3. Test data and scenarios are realistic and representative of production operational conditions, including normal operation, fault conditions, and edge cases.
4. Traceability matrix confirms that all integration requirements from SK-D07-002 through SK-D07-006 have corresponding SIT test cases.
5. Test environment setup is documented and can be reproduced to support test execution and later operational validation.
6. SIT protocol has been reviewed and approved by at least the Security Technical Consultant, Test Engineer, and Operations Engineer.

## Composition Patterns

SIT protocol development follows a sequential composition with SK-D07-002 through SK-D07-006, which define the integration requirements that SIT must verify. SIT test execution (not covered by this skill) occurs after design completion (R2) and guides implementation verification (R3).

## Source Traceability

- IEC 62443-1-1 defines the overall IACS security assurance framework, which requires system-level verification testing at appropriate lifecycle stages.
- IEC 62443-4-2 specifies technical requirements for IACS components; SIT verifies that integrated components meet these requirements collectively.
- IEC 62443-3-3 defines R3 system security requirements including secure communication, access control, and audit logging; SIT verifies these are enforced in the integrated environment.
- IEEE 829 Standard for software test documentation provides structured templates for test plan and test case documentation.
- ISTQB Testing Standards provide methodologies for black-box and integration testing applicable to SIT scenarios.
- ID14 exemplar or site-specific test frameworks provide domain-specific test categories and proven test case templates for SIT integration scenarios.
