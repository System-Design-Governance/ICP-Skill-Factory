## Description

DER Aggregation Strategy Design is the systematic design of Distributed Energy Resource (DER) aggregation architectures for Virtual Power Plant (VPP) operations, defining how individual DER assets (photovoltaic generators, battery energy storage, controllable loads, electric vehicles) are grouped, prioritized, dispatched, and coordinated through Distributed Energy Resource Management System (DERMS) and Energy Management System (EMS) platforms. The skill encompasses aggregation topology, dispatch logic, communication architecture security, and contingency response strategies that enable coordinated DER operation as flexible grid resources.

## Description — Scope

This skill encompasses the design of logical and physical aggregation architectures for DER assets, definition of aggregation groupings based on network location and functional characteristics, design of dispatch priority logic for simultaneous control of diverse DER types, specification of DERMS-to-DER control communication paths and cybersecurity requirements, and design of fallback/safety strategies when control communication fails. The skill does not include detailed power dispatch optimization algorithms—those are addressed in SK-D05-004. The skill boundary excludes individual DER sizing and design, which is addressed in SK-D03-001 through SK-D03-005, and excludes detailed network communication system design which is addressed in SK-D07-001.

## Acceptance Criteria

- DER aggregation groupings defined: logical assignment of individual DER assets to aggregation groups based on network location, functionality, and coordinated control requirements
- Aggregation topology documented: hierarchical structure showing primary aggregator (DERMS), secondary aggregators (if applicable for large deployments), and DER assets within each group
- Dispatch priority logic designed: algorithm for sequencing dispatch commands when simultaneous control of multiple DER types required (PV curtailment, BESS discharge, load shedding)
- DERMS-to-DER control communication architecture specified: communication protocol, data points required for dispatch (setpoint command, permission status, telemetry feedback), command latency requirements
- Cybersecurity requirements specified: authentication for DERMS-to-aggregator and aggregator-to-DER communication, encryption of control commands, integrity checks for dispatch data, audit logging
- Fallback strategies designed: automatic actions when DERMS communication lost (e.g., BESS discharges to specified SOC, PV operates at reduced power factor, loads shed per priority list)
- Grid services capability documented: aggregation capacity for frequency regulation, volt-var support, peak shaving, energy arbitrage, and reactive power support
- Contingency response procedures designed: aggregation response to grid frequency deviations, voltage disturbances, and generation loss events
- Performance metrics defined: aggregation responsiveness (time from command to 90% power change), aggregation accuracy (actual vs. commanded power), steady-state accuracy

## Standards

DER aggregation strategy design is critical to IEC 62443 security in distributed energy systems. The DERMS platform and DER-to-aggregator communication links are high-value targets for attack that could compromise grid stability if compromised (SL-D resilience objective). The aggregation strategy must include redundant communication paths, cryptographic security for control command authentication, and fallback strategies that automatically enforce safe operating constraints when supervisory control is unavailable. The aggregation architecture enables implementation of critical protective controls (frequency-based load shedding, voltage-based reactive power compensation) that must be protected from unauthorized modification or delay.

- Contingency analysis and grid stress scenarios

## Description — Process Steps

1. Inventory all DER assets: location, type (PV, BESS, controllable load, EV), capacity, control capability, communication interface
2. Define aggregation groupings: assign assets to primary aggregation groups based on network location, functionality, and coordinated control requirements
3. Create aggregation topology diagram: show DERMS, primary/secondary aggregators, aggregation groups, individual DER assets
4. Define dispatch priority logic: algorithm for commanding multiple asset types with different response characteristics (PV fast curtailment, BESS moderate ramp, load shedding slower response)
5. Specify DERMS-to-DER communication paths: protocol selection, data points (setpoint, status, telemetry), update frequency, command latency requirement
6. Identify cybersecurity requirements: authentication protocol for DERMS command authorization, encryption for sensitive data, integrity protection for dispatch commands, audit trail for all commands
7. Design fallback strategies: automatic actions when DERMS communication lost (time-based fallback, SOC-based fallback, frequency-based load shedding)
8. Model aggregation response: transient response simulation to frequency deviations, voltage disturbances, generation loss
9. Define grid services capability: calculate aggregation power reserve, ramp rate, and responsiveness for each service (frequency regulation, volt-var, peak shaving)
10. Design contingency procedures: aggregation response to RoCoF events, over-frequency events, under-frequency events, loss of utility grid
11. Specify performance monitoring: define metrics for command execution latency, setpoint accuracy, aggregation availability
12. Document security architecture: threat assessment, security controls, key rotation procedures, incident response

## Description — Decision Logic

DER aggregation strategy design results guide critical system architecture decisions:
- Aggregation grouping: single large aggregation vs. multiple smaller aggregations affects control complexity and resilience
- Communication protocol: cellular vs. broadband powerline vs. dedicated fiber affects latency, cost, and cybersecurity exposure
- Dispatch priority: PV curtailment vs. BESS discharge vs. load shedding sequencing affects customer experience and grid support services
- Fallback strategy: aggressive fallback (immediate load shedding) vs. conservative fallback (preserve load but risk grid instability)
- Hierarchical structure: flat aggregation vs. multi-level aggregation affects scalability and fault isolation
- Cybersecurity architecture: centralized encryption key vs. distributed key management affects key rotation complexity

## Description — Common Errors

- Assuming aggregation communication latency can be ignored: 100ms latency is critical for frequency response
- Treating all DER asset response times as equivalent: PV ramps in seconds, BESS in milliseconds, load in minutes
- Neglecting fallback strategy testing: assuming DERMS control always available creates grid vulnerability
- Designing aggregation without considering communication failures: single communication path creates single point of failure
- Underestimating cybersecurity risk in control architecture: open authentication creates denial-of-service vulnerability
- Assuming DER assets can be optimally dispatched without considering individual asset constraints: BESS SOC limits, PV power factor limits
- Treating aggregation as purely market optimization problem without considering grid stability constraints

## Acceptance Criteria — Quality Metrics

- Aggregation grouping justification: network location and functional characteristics documented for each group
- Communication architecture verification: protocol latency measured, jitter quantified, packet loss handling specified
- Dispatch priority logic testing: simulation results showing correct sequencing for simultaneous multi-asset commands
- Cybersecurity architecture review: threat modeling completed, security controls verified for each threat, penetration testing recommended
- Fallback strategy testing: simulated DERMS communication loss showing automatic fallback enforcement
- Performance metrics: aggregation command latency <100ms for frequency response, setpoint accuracy ±3%, aggregation availability >99%
- Grid services capability: aggregated reserve capacity and ramp rates documented for each service
