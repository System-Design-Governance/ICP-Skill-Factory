## Description

This skill encompasses the evaluation, selection, and configuration of time-series database platforms suitable for storing operational data from OT/ICS systems including SCADA, historian, and telemetry data. Time-series databases (InfluxDB, TimescaleDB, OSIsoft PI, AVEVA Historian, Prometheus) are optimized for high-volume, time-stamped data insertion and efficient historical data queries. The skill covers database architecture assessment, platform comparison and selection based on project requirements, sizing calculations for expected data volumes and retention periods, configuration for optimal performance and reliability, replication and high-availability design, and security hardening including access control and encryption. The selected database becomes a critical infrastructure component for data analytics, compliance reporting, and security monitoring per SK-D12-004 (Data Quality Monitoring) and SK-D12-006 (Analytics Dashboard Design).

## Inputs

- Data Acquisition Architecture specification (from SK-D12-001: Data Acquisition Architecture) — data types, frequencies, and volumes to be stored
- Data Retention Policy (from SK-D12-004: Data Retention Policy Design) — retention periods drive database sizing calculations
- System Architecture and Infrastructure design (from SK-D02-001: OT Network Topology Design) — physical and logical infrastructure constraints
- Security requirements and access control policies (from SK-D01-020: Account and Access Control; SK-D01-033: Data Classification Policy)
- High Availability and Disaster Recovery requirements (from project quality and reliability targets)
- Organizational technology standards and vendor preferences
- Budget and cost constraints for infrastructure

## Outputs

- Database Selection Matrix: comparison of candidate time-series database platforms against evaluation criteria (performance, scalability, cost, vendor support, security features)
- Database Sizing Calculations: estimated storage requirements based on data volumes, retention periods, and compression; hardware specifications (CPU, memory, disk) recommended
- Database Configuration Documentation: detailed specifications for deployment including replication settings, retention policies, query optimization parameters, backup strategies
- High Availability Design Document: cluster configuration, failover procedures, recovery time objectives (RTO), and recovery point objectives (RPO)
- Security Configuration Checklist: database access control (user authentication, RBAC), encryption at rest and in transit, audit logging, and compliance with IEC 62443-3-3
- Performance Testing Plan: benchmarks for query response times, data insertion rates, and system scalability validation

## IEC 62443 Lifecycle Stages

- R2 (Detailed Design) — database platform selection and design specifications are finalized
- R3 (Implementation) — database infrastructure is procured, deployed, and configured
- R4 (Commissioning/Startup) — database is tested and optimized for production data volumes during commissioning
- R5 (Operations) — database is maintained, monitored, and scaled as data volumes grow

## Acceptance Criteria

1. Database selection is documented in a Selection Matrix comparing at least 3 candidate platforms against defined evaluation criteria (performance, cost, vendor support, security, scalability); rationale for selected platform is clearly stated
2. Sizing calculations are based on projected data volumes (GB/day or records/second), retention periods, and compression ratios; recommended hardware specifications (CPU, memory, storage) are documented with assumptions clearly stated
3. Database configuration documentation specifies replication settings, failover mechanisms, retention policy enforcement, query optimization parameters, and backup/recovery procedures with RTO and RPO targets
4. High Availability design provides for automatic failover with documented procedures; cluster configuration and failover testing methodology are specified
5. Security configuration checklist confirms: user authentication, role-based access control (RBAC), encryption at rest and in transit, audit logging of all data access, and alignment with IEC 62443-3-3 security requirements
6. Database design has been reviewed and approved by SYS, SAC, and a database specialist; performance assumptions have been validated through testing or reference deployments
