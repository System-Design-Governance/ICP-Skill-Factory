## Description

Concept Zone/Conduit Architecture development is the lightweight, advisory predecessor to the full Zone/Conduit Architecture Design skill (SK-D01-001). This skill guides the creation of conceptual (non-binding) network segmentation and communication conduit architectures during the Pre-Gate 0 phase. Per IEC 62443-1-1 §6.5.1.1 and GOV-SD governance, concept architecture is one of five mandatory Gate 0 information package inputs and carries explicit non-binding status. This skill validates architectural feasibility against customer requirements (SK-D14-001 ⏳) and informs proposal design without committing to detailed implementation specifications.

## Inputs

- Customer operational requirements and constraints from SK-D14-001 ⏳
- Site survey findings and network topology from SK-D14-011 ⏳
- Infrastructure inventory and discovery data from SK-D14-012 ⏳
- Security classification and protection profile from SK-D14-014 ⏳
- Regulatory and standards alignment requirements
- Customer risk appetite and resilience targets
- High-level functional architecture diagrams

## Outputs

- Concept Zone/Conduit architecture diagram (logical, non-binding)
- Zone/Conduit mapping to functional/operational requirements
- Preliminary security control allocation to zones
- Communication conduit specifications (bandwidth, latency, reliability requirements)
- Architectural assumptions and dependencies documentation
- Alignment statement to IEC 62443 foundational requirements (purview, lifecycle, etc.)
- Feasibility commentary for proposal and Gate 0 decision support

## Roles

- PGS (Pre-Gate Design Support): Leads concept architecture development, coordinates with customer, ensures alignment with governance
- Systems Architect: Develops Zone/Conduit concept, evaluates feasibility, documents assumptions
- OT Subject Matter Expert: Validates operational feasibility and protection requirements
- Governance Officer: Confirms alignment to GOV-SD mandatory inputs and non-binding guidance

## Acceptance Criteria

1. Concept Zone/Conduit architecture diagram includes at minimum 3 security zones or DMZ topologies
2. Each zone is mapped to customer operational requirements from SK-D14-001 ⏳
3. Communication conduit specifications include required bandwidth, latency, and resilience parameters
4. Security control allocation to zones is documented with rationale
5. Architecture documentation carries explicit "non-binding concept" label and disclaimer
6. Architectural assumptions are listed and validated against site survey findings
7. Alignment statement confirms that concept architecture does not preclude detailed design changes in Gate 1+
8. Feasibility assessment identifies preliminary technical risks and architectural constraints
9. Gate 0 information package includes concept architecture per GOV-SD mandatory inputs
