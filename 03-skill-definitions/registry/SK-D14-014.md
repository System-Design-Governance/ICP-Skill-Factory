## Description

Perform the preliminary security classification for the SuC, establishing initial Security Level Target (SL-T) proposals per system zone and an overall project security risk classification that feeds into the Gate 0 decision package. This skill bridges the gap between customer security requirements (from SK-D14-010) and the formal risk assessment process (SK-D01-006) by providing an early, indicative security classification before the project is formally approved. Under GOV-SD, preliminary security classification informs Gate 0's scope stability assessment — stakeholders need to understand the security ambition level before committing resources. The boundary of this skill is preliminary, indicative classification for pre-project decision-making; the formal SL-T assessment with full zone analysis falls under SK-D01-010 ⏳, and the detailed risk assessment falls under SK-D01-007.

## Inputs

- Customer/employer security requirements (from SK-D14-010)
- General requirements specification (from SK-D14-001)
- Concept Zone/Conduit architecture (from SK-D14-013 ⏳) — if available at this stage
- Existing infrastructure inventory (from SK-D14-012) — brownfield security posture baseline
- Site survey findings (from SK-D14-011) — physical security constraints
- ID03 §5.4.1: Risk classification methodology
- ID01 §6.5.1.2: Systems security classification guidance
- Industry sector security baselines (sector-specific SL-T norms)

## Outputs

- Preliminary Security Classification Report:
  - Project security risk classification: overall risk category (High / Medium / Low) with justification
  - Proposed SL-T per zone: initial Security Level Target assignments based on customer requirements, industry norms, and preliminary threat landscape assessment
  - Classification rationale: per-zone justification linking SL-T to customer requirements, industry sector norms, and threat exposure
  - Scope implications: how the proposed SL-T levels affect project scope, effort, and cost (higher SL = more controls = more effort)
  - Uncertainty assessment: confidence level of preliminary classification, key assumptions, and factors that may change during formal R1 assessment
- Preliminary Zone Map: high-level zone identification (not yet formal Zone/Conduit architecture) showing major security domains and their proposed SL-T
- Security Ambition Summary: one-page executive summary of security classification for Gate 0 decision-makers

## Automation Potential

**Level:** Partial

Industry sector SL-T benchmarking can be partially automated (lookup tables for common sectors). SL-T scope impact calculation can be automated (SR count per SL level, effort multipliers). However, risk classification requires expert judgment (threat landscape assessment for specific project context), SL-T assignment requires balancing multiple factors (customer expectations, achievability, cost), and uncertainty assessment requires experienced evaluation of classification confidence.

## Acceptance Criteria

1. Project security risk classification assigned (High/Medium/Low) with documented justification referencing customer requirements and industry context
2. Proposed SL-T assigned per identified zone with per-zone rationale linking to specific customer requirements or industry norms — no zones without SL-T proposal
3. Scope implications documented: how proposed SL-T levels affect project effort, cost, and timeline — quantified where possible (e.g., SL-3 vs. SL-2 effort delta)
4. Uncertainty assessment included: confidence level stated, key assumptions listed, and factors that may change during formal R1 assessment identified
5. Security ambition summary produced: one-page executive-ready document suitable for Gate 0 decision-makers
6. Classification marked as preliminary/indicative per GOV-SD — explicitly states that formal SL-T confirmation occurs at Gate 1 (SL Decision Lifecycle)

## Estimated Effort

| Level | Effort | Notes |
|-------|--------|-------|
| Junior (< 2 yr) | 3–5 person-days | Assumes medium-complexity project, ~5 zones; includes analysis, SL-T proposal, scope impact, and report writing |
| Senior (5+ yr) | 2–3 person-days | Same scope; senior leverages sector SL-T benchmarks and rapid classification patterns |

Notes: Effort is relatively low as this is a preliminary assessment, not a formal risk analysis. Complexity increases with zone count and unfamiliar industry sectors. Projects with explicit customer SL requirements are faster (mapping exercise); projects with vague security expectations require more interpretive effort. This classification will be superseded by formal SL-T assessment (SK-D01-010 ⏳) during R1.
