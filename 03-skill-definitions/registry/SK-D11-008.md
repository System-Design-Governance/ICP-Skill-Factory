## Description — Purpose & Scope
Manage the complete lessons learned lifecycle encompassing collection during project execution, categorization (technical, process, commercial), root cause analysis, knowledge extraction, and strategic dissemination to future projects. This skill ensures organizational learning is systematically captured and leveraged to improve future project delivery and operational practices.

## Roles
- Establish and maintain lessons learned collection procedures and governance
- Facilitate lessons learned workshops at project milestones and completion
- Categorize captured lessons (technical, process, commercial, organizational)
- Conduct root cause analysis for significant lessons and issues
- Extract actionable insights and knowledge from collected lessons
- Manage and maintain lessons learned database
- Ensure knowledge dissemination to relevant future projects
- Integrate lessons learned with SK-D11-009 (Technical Knowledge Base)
- Track implementation of recommended actions across projects
- Report on organizational learning metrics and improvements

## Roles — Competency Requirements
- Expertise in knowledge management systems and databases
- Strong facilitation and workshop leadership skills
- Advanced analytical capabilities for root cause analysis
- Understanding of categorization frameworks and taxonomies
- Experience with lessons learned collection and dissemination processes
- Proficiency in knowledge transfer methodologies
- Project management and organizational development knowledge
- Ability to identify patterns across multiple projects

## Acceptance Criteria
- Systematic lessons learned collection at all project phases
- 100% of significant issues documented with root cause analysis
- Lessons categorized and indexed in searchable knowledge system
- Documented linkage between lessons and SK-D11-009 knowledge base entries
- Minimum 80% project team participation in lessons learned workshops
- Measurable reduction in recurring issues across projects (year-over-year)
- Demonstrated implementation of lessons in subsequent projects
- Regular reporting on organizational learning progress and impact
- Integration with SK-D11-012 training program development

## Dependencies
- Project milestone review and lessons collection
- Lessons learned workshop facilitation and documentation
- Root cause analysis and impact assessment
- Knowledge categorization and database management
- Knowledge dissemination and stakeholder communication
- Implementation tracking and effectiveness measurement
- Continuous improvement monitoring

## Dependencies — Integration Points
- **Upstream**: Project execution, issue tracking, performance data
- **Downstream**: SK-D11-009 (Knowledge Base), SK-D11-012 (Training Program)
- **Lateral**: Quality assurance, process improvement, organizational development

## Description — Risk Mitigation
- Lessons collection becoming bureaucratic burden reducing participation
- Root cause analysis lacking rigor or accuracy
- Knowledge silos preventing cross-project dissemination
- Difficulty demonstrating ROI of lessons learned investment
