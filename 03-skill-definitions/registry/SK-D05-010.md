## Description

Configure IEC 61850 Substation Configuration Language (SCL) files and digital substation integration for protection relays, merging units, SCADA servers, and network switches in a power generation or distribution system. This skill covers design and generation of three main SCL file types: IED Capability Descriptions (ICD) for individual intelligent electronic devices (relays, meters), System Configuration Descriptions (SCD) defining the complete system topology and signal exchanges, and configuration of GOOSE (Generic Object Oriented Substation Event) control blocks for high-speed protection signaling, sampled value (SV) streams for real-time analog data, and MMS (Manufacturing Message Specification) report control blocks for SCADA integration. SCL configuration directly implements protection logic from SK-D04-004 (Advanced Protection Schemes), enables digital substation integration, and ensures interoperability between multivendor devices. This skill is critical for modern digital substations and smart grid environments and feeds into system commissioning and testing.

## Inputs

- Approved protection logic and schemes (from SK-D04-004: Advanced Protection Schemes)
- IED Capability Descriptions (ICD) from device vendors (relays, merging units, bay controllers)
- System architecture and signal requirements (from SK-D02-004 ⏳: Data Flow Diagram Development)
- GOOSE signal specifications: priority, integrity checking requirements (from SK-D04-001, SK-D04-003)
- Sampled value stream requirements: current/voltage signals, reporting intervals (from SK-D04-002 ⏳: Protection Coordination Design)
- Zone/Conduit security boundaries and authentication/encryption policies (from SK-D01-001: Zone/Conduit Architecture Design)
- Network topology and switch configurations (VLAN requirements from SK-D02-001 ⏳: OT Network Topology Design)
- Cybersecurity requirements per IEC 62443 (from SK-D01-003 ⏳: Cybersecurity Control Selection)

## Outputs

- System Configuration Description (SCD) file (.xml per IEC 61850-5-92): complete system topology, all devices, communication bindings
- IED Configuration Files: device-specific GOOSE control blocks, sampled value streams, MMS access lists
- GOOSE Signal Assignment Table: signal ID, source device, destination device, priority, latency requirement, integrity checking method
- Sampled Value Configuration: channel specifications, reporting intervals, quality indicators
- MMS Report Configuration: report control blocks, trigger conditions, buffering strategy
- SCL Validation Report: schema validation, consistency checks across SCD
- Digital Substation Integration Plan: GOOSE/SV network design, VLAN assignments, switch timing synchronization
- Security Configuration Specification: IEC 62443 authentication/encryption mappings to SCL security parameters

## Roles

- DES (Design Engineer) — primary: designs and generates SCL files and GOOSE/SV signal mappings
- STC (Security Technical Consultant) — reviews security aspects (encryption, authentication, access control in SCL)
- SYS (System Architect) — provides system topology and performance constraints
- PRL (Protection Specialist) — validates that protection logic is correctly mapped to GOOSE/MMS signals

## Dependencies

- SK-D02-004 ⏳: Data Flow Diagram Development — signal exchange requirements and frequencies are the source of truth
- SK-D02-001 ⏳: OT Network Topology Design — physical network topology and VLAN design required

### Soft Dependencies (Enhances Effectiveness)

- SK-D04-001: Overcurrent Protection Settings — setting values feed into GOOSE control block thresholds
- SK-D05-008 ⏳: SCADA/RTU Configuration Development — MMS configuration in SCL must align with SCADA data model

## Acceptance Criteria

1. SCD file passes IEC 61850-5-92 schema validation with zero errors
2. All protection logic signals (overcurrent, distance, differential, etc.) from SK-D04-004 are mapped to GOOSE control blocks with documented signal IDs
3. Every GOOSE signal has a defined priority, latency requirement, and integrity checking method
4. Sampled value streams are configured for all required current and voltage inputs; reporting intervals match protection scheme requirements (typically ≤20 ms per IEC 61850-9-2)
5. MMS report control blocks are configured with appropriate trigger conditions and buffering strategy for SCADA integration
6. Network VLAN assignments and switch timing are specified to meet GOOSE communication latency requirements (typically ≤4 ms end-to-end)
7. Cybersecurity parameters (encryption, authentication) in SCD match Zone/Conduit conduit specifications from SK-D01-001
8. Multivendor device interoperability is confirmed through compatibility matrix or pilot testing
9. SCD file is approved by DES and reviewed by STC for security and PRL for protection logic compliance
