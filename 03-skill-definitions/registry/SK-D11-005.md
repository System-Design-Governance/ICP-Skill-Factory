## Description — Purpose & Scope
Analyze engineering process efficiency to identify bottlenecks, waste, and improvement opportunities across project delivery workflows. This skill feeds directly into SK-D11-004 (Engineering SOP Development) to implement identified process improvements and optimize organizational delivery capabilities.

## Roles
- Conduct comprehensive process mapping of engineering workflows
- Perform detailed cycle time analysis to identify delays and inefficiencies
- Execute value stream mapping to visualize material and information flow
- Identify process bottlenecks, redundancies, and waste sources
- Develop data-driven improvement recommendations
- Benchmark against industry best practices and internal baselines
- Calculate efficiency metrics and performance indicators
- Create visual process models and documentation

## Roles — Competency Requirements
- Expertise in process analysis methodologies (Lean, Six Sigma)
- Advanced facility with process mapping tools and techniques
- Strong analytical and quantitative skills
- Understanding of engineering project delivery workflows
- Experience with value stream and cycle time analysis
- Proficiency in process visualization and documentation
- Knowledge of process improvement frameworks

## Acceptance Criteria
- Complete process maps documenting current-state workflows
- Identified bottlenecks with root cause analysis
- Quantified cycle time reduction opportunities
- Value stream analysis with waste elimination strategies
- Prioritized improvement recommendations with expected impact estimates
- Baseline metrics and efficiency measurements established
- Clear handoff to SK-D11-004 with actionable improvement recommendations
- Stakeholder alignment on priority improvement initiatives

## Dependencies
- Engineering workflow assessment and analysis
- Data collection and process performance measurement
- Bottleneck identification and root cause investigation
- Process redesign recommendation development
- Efficiency baseline establishment
- Improvement opportunity prioritization

## Dependencies — Integration Points
- **Upstream**: Process observation and current-state documentation
- **Downstream**: SK-D11-004 (SOP Development) implementation of improvements
- **Lateral**: Quality management, resource planning, scheduling

## Description — Risk Mitigation
- Analysis paralysis delaying implementation actions
- Recommendations based on incomplete process understanding
- Change resistance during process improvement rollout
- Resource constraints limiting analysis depth
