## Description

This skill encompasses the selection of electrical and electronic components for operational technology (OT) and industrial control systems (ICS) panel assemblies, coupled with the authoring of comprehensive component specification sheets. Component selection criteria integrate technical performance, cybersecurity alignment with IEC 62443, supply chain resilience, and lifecycle support requirements. Specification documentation captures electrical ratings, environmental operating conditions, cybersecurity considerations, and vendor/part number cross-references to ensure design consistency and procurement compliance.

---

## Description — Business Context

Component selection and specification writing is a foundational design activity within secure system assembly. Proper component selection ensures that OT/ICS assemblies meet operational requirements while maintaining cybersecurity posture aligned with IEC 62443 standards. This skill bridges design intent with procurement and manufacturing execution, enabling downstream activities including panel layout, wiring diagram development, and component bill of materials (CBOM) compilation.

## Description — Procedural Steps

**Phase 1: Component Requirement Definition**
1. Extract component-level requirements from DRS and system architecture
2. Define electrical ratings (voltage, current, power, frequency)
3. Specify environmental operating envelope (temperature, humidity, pollution degree per IEC 60721-3)
4. Identify communication interface requirements (Modbus, Ethernet, serial, wireless)
5. Document cybersecurity attribute requirements per IEC 62443 (e.g., secure boot, firmware update mechanism)

**Phase 2: Component Selection**
1. Research candidate components matching identified requirements
2. Evaluate against selection criteria:
   - Operational environment compatibility (derating curves, thermal analysis)
   - IEC 62443 compliance: verify component supports required security functions
   - Supply chain availability: check lead time, production volume, geographic sourcing
   - Lifecycle support: manufacturer product road-map, end-of-life projections
   - Interoperability: mechanical envelope, electrical interface, protocol compatibility with existing infrastructure
   - Cost-benefit trade-offs: performance vs. cost vs. risk
3. Document candidate component evaluation matrix
4. Select preferred component and identify qualified alternatives (for supply chain resilience)

**Phase 3: Specification Sheet Authoring**
1. Create specification sheet template with: component designation, functional description, electrical ratings, environmental limits, mechanical dimensions, cybersecurity attributes
2. Document electrical ratings: voltage/current operating range, power dissipation, thermal characteristics
3. Specify environmental requirements: operating temperature range, humidity, altitude, vibration immunity per relevant IEC standards
4. Record cybersecurity considerations: firmware update mechanism, secure communications support, debug interface security, supply chain traceability
5. Include vendor/part number cross-references and approved substitutes
6. Define acceptance criteria for component receiving inspection and testing
7. Obtain design engineer and procurement sign-off

**Phase 4: Design Integration**
1. Integrate component selections into system schematic and functional block diagram
2. Verify no conflicts or incompatibilities introduced
3. Update Bill of Materials (BOM) with component selection decisions
4. Distribute specification sheets to downstream teams (manufacturing, procurement, quality)

## Acceptance Criteria

- All components selected meet specified electrical ratings, environmental requirements, and IEC 62443 security attributes
- Specification sheets document: part number, functional description, electrical ratings, environmental limits, cybersecurity attributes, vendor cross-references, and acceptance criteria
- Component selections validated against interoperability matrix with existing infrastructure
- Supply chain risk assessment completed: no single-source critical components without approved alternatives identified
- Design engineer and procurement approval obtained on component specification sheets
- Traceability established: each component traces to design requirement and forward to CBOM

---

## Description — Common Pitfalls

**Pitfall: Over-specification or under-specification of environmental requirements**
- *Mitigation:* Cross-check operating environment profile with site data; apply appropriate derating factors to component ratings

**Pitfall: Selecting components lacking cybersecurity support required by IEC 62443 target SL**
- *Mitigation:* Pre-screen candidate components for security attributes; document justification if security-capable alternative unavailable

**Pitfall: Insufficient supply chain contingency planning**
- *Mitigation:* Identify qualified alternatives for critical components; assess geographic sourcing concentration

**Pitfall: Incomplete or ambiguous specification sheets causing procurement/manufacturing delays**
- *Mitigation:* Use standardized specification template; include diagram, electrical curves, and acceptance criteria examples
