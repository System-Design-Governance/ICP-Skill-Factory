## Description

Design Human-Machine Interface (HMI) screens for SCADA/EMS/DERMS operator workstations following ISA-101 HMI design guidelines. This skill covers screen hierarchy and navigation structure, color coding standards for operational and alarm states, alarm visualization techniques, and situational awareness displays optimized for different operator roles and operational contexts. Design includes ergonomic layout, information density optimization, and integration with role-based access controls. Security considerations include screen-level role-based access per SK-D01-020 (Role-Based Access Control Design) and integration with audit logging per SK-D01-019 (Audit Trail Design).

## Inputs

- System Architecture Design and SCADA/EMS/DERMS functional specification (from SK-D05-001 ⏳ and SK-D05-002 ⏳)
- Process flow diagrams and operational procedures (from SK-D05-001 ⏳: Process Flow Mapping)
- Operator role profile analysis with permission matrix (from SK-D01-020: Role-Based Access Control Design)
- Alarm hierarchy and priority definitions (from SK-D05-006: Alarm Hierarchy and Configuration Design)
- Device network topology and real-time data availability (from SK-D02-001 ⏳: OT Network Topology Design)
- Operator workstation hardware specifications (monitors, input devices, ambient conditions)
- Existing HMI standards or customer branding guidelines

## Outputs

- HMI Screen Specification Document: screen templates for each operational view (overview dashboard, zone control, generator/asset control, alarms, trending, reports)
- Screen Layout Diagrams (mockups or wireframes) showing navigation hierarchy, element placement, and role-specific variants
- Color Palette and Coding Standard document (per ISA-101 Section 5.3): normal state colors, alarm colors (red/yellow/blue), trends, process states
- Navigation Map: screen relationships, drill-down paths, context switching rules, role-based screen visibility matrix
- HMI Object Library: standard symbols, gauges, trend displays, alarm displays with interaction rules
- Integration Matrix: mapping HMI screen elements to backend data tags, control objects, and command routes per SK-D05-002 ⏳

## Tools

- Figma / Adobe XD / Sketch (modern HMI/UX design tools with prototyping)
- MATLAB/Simulink Stateflow (for complex visualization logic)
- SQL/structured markdown (for screen–data mapping tables)
- Graphviz / D2 (for screen navigation diagrams)
- ISA-101 reference material and HMI design pattern libraries

## Standards

- ISA-101:2015 — Criterion 5 (Human-Machine Interfaces): screen hierarchy, color coding, alarm presentation, operator guidance
- ISA-82.00.01 — Graphic Symbols for Process Displays: standard symbols for equipment, processes, and control elements
- IEC 60417 — Graphical Symbols for Use on Equipment: general-purpose symbols for HMI screens
- ANSI/ISA-18.2 — Standard for Alarm Management for the Process Industries: alarm display and acknowledgment interaction patterns
- ISO 11064 — Ergonomics of Human-System Interaction: control room design, workstation layout, and visual ergonomics
- IEC 62443-3-3 — System Security Requirements and Security Levels: security presentation and operator awareness of security status

## IEC 62443 Lifecycle Stages

- R2 (Detailed Design — HMI screens designed to detailed specification)
- R3 (Implementation — HMI coding and testing against approved specifications)

## Roles

- DES (Design Engineer) — primary: creates screen layouts, navigation flow, visual design
- HMI/SCADA Specialist — supports with operational workflow optimization and alarm presentation best practices
- STC (Security Technical Consultant) — reviews screen design for security information leakage and sensitive data handling
- Operator Representative — validates usability and situational awareness effectiveness
- SAC (Security Architect) — ensures role-based screen access and audit logging integration

## Dependencies

### Hard Dependencies (Required Before Execution)

- SK-D05-002 ⏳: SCADA/EMS/DERMS Functional and Software Design — cannot design HMI screens without knowing data model, available functions, and control commands
- SK-D05-006: Alarm Hierarchy and Configuration Design — alarm display design depends on priority hierarchy and alarm rules
- SK-D01-020: Role-Based Access Control Design — role definitions and permissions must be finalized before designing role-specific screen variants

### Soft Dependencies (Enhances Effectiveness)

- SK-D02-001 ⏳: OT Network Topology Design — network topology informs data availability and screen refresh rates
- SK-D01-019: Audit Trail Design — audit logging integration patterns inform audit trail screen design
- SK-D05-002 ⏳: Process Flow Mapping — process workflows inform operational screen hierarchy and navigation
- SK-D05-003 ⏳: Operational Procedures Development — operating instructions inform contextual help and operator guidance on screens

## Acceptance Criteria

1. All major operational views (overview, zone/asset control, alarms, trending, reports, diagnostics) are designed with screen mockups or wireframes
2. Screen navigation hierarchy is documented with state diagrams showing drill-down and return paths
3. Color palette follows ISA-101 Section 5.3 specifications for normal/alert/alarm/disabled states, with WCAG AA contrast compliance
4. Alarm display design includes priority color coding, acknowledgment affordance, and alert suppression indicators per ISA-18.2
5. Role-based screen visibility matrix is complete and traceable to SK-D01-020 permissions
6. HMI-to-data-tag mapping is complete and validated against SK-D05-002 ⏳ data model
7. Screen layout meets ergonomic guidelines per ISO 11064 for typical operator workstation (e.g., 24" dual monitors, 80 cm viewing distance)
8. Design documentation includes design rationale for screen grouping, information density, and navigation paths
9. Design has been reviewed and approved by DES, operator representative, and STC
