## Description

Conduct on-site survey of the project location to assess physical environment, infrastructure constraints, and operational conditions that impact system design, and document all findings in a structured constraint register. Site survey outputs directly feed into feasibility assessment (SK-D14-003 ⏳) and concept architecture design (SK-D14-013 ⏳) by identifying real-world constraints that cannot be determined from documents alone. This skill covers the physical site assessment and constraint documentation; the inventory of existing infrastructure equipment falls under SK-D14-012, and the technical feasibility evaluation that consumes these constraints falls under SK-D14-003 ⏳.

## Inputs

- Customer/employer site access arrangements and safety requirements
- Project scope and general requirements (from SK-D14-001)
- Available site documentation (floor plans, cabinet layouts, cable routes, electrical diagrams)
- Site survey checklist (physical, environmental, network, safety categories)
- Previous site survey reports (if brownfield with prior work)

## Outputs

- Site Survey Report:
  - Physical environment assessment: building layout, room dimensions, cabinet space availability, floor loading capacity, cable routing options
  - Environmental conditions: temperature/humidity ranges, dust/contamination levels, vibration exposure, EMI sources
  - Electrical infrastructure: power supply availability, UPS capacity, grounding quality, cable tray capacity
  - Network infrastructure: existing cable plant, fiber routes, wireless coverage, patch panel availability
  - Physical security: access control to equipment rooms, CCTV coverage, perimeter security
  - Safety constraints: hazardous area classifications, safety system zones (SIS boundaries), emergency shutdown paths
  - Photographic evidence: annotated site photographs documenting key constraints
- Constraint Register: structured table of all identified constraints with: constraint ID, category, description, severity (Hard / Soft / Informational), affected design areas, and mitigation options
- Site Access Requirements Document: logistics information for subsequent site visits (access procedures, safety induction, escort requirements, work hours)

## Tools

- Site survey checklist templates (printed/tablet forms)
- Measurement tools (tape measure, laser distance meter, thermal imager)
- Camera/documentation tools (annotated photo capture)
- Network discovery tools (cable tester, fiber light source — for infrastructure verification)
- Environmental monitoring tools (temperature/humidity logger)
- CAD tools (for floor plan annotation)

- DES (Design Engineer) — supports physical infrastructure assessment, captures measurements

## Dependencies

### Hard Dependencies (Required Before Execution)

- None — site survey can be conducted as soon as site access is arranged, typically among the first Pre-Gate activities

### Soft Dependencies (Enhances Effectiveness)

- SK-D14-001: Requirements Specification Development — understanding project scope focuses the survey on relevant areas
- SK-D14-012: Existing Infrastructure Inventory — often conducted simultaneously during the same site visit

## Automation Potential

**Level:** Human-Only

Site survey is fundamentally a physical, on-site activity requiring human presence, observation, and judgment. Photographic documentation and constraint register population can be streamlined with tablet-based forms, but the core activity — assessing physical conditions, identifying constraints, and evaluating design implications — requires human expertise and physical presence. No meaningful automation of the survey itself is possible.

## Acceptance Criteria

1. Site survey report covers all six assessment categories: physical, environmental, electrical, network, physical security, and safety — no category skipped without documented rationale
2. Constraint register produced with minimum 10 documented constraints (for a typical single-site project) — each with unique ID, severity classification, and affected design area
3. Photographic evidence captured for all Hard severity constraints — annotated photos showing the constraint and its context
4. Environmental conditions documented with measured values (temperature range, humidity) — not just qualitative descriptions
5. Site access requirements documented for subsequent visits — logistics information sufficient for project team to plan future site work
6. Survey findings directly usable as input to feasibility assessment (SK-D14-003 ⏳) — constraint register format compatible with feasibility evaluation framework

## Estimated Effort

| Level | Effort | Notes |
|-------|--------|-------|
| Junior (< 2 yr) | 3–5 person-days | Assumes single-site survey, 1 day on-site + 2–4 days documentation and report writing |
| Senior (5+ yr) | 2–3 person-days | Same scope; senior conducts efficient focused survey with rapid documentation |

Notes: Multi-site projects require one survey per site. Remote/international sites add travel time. Large industrial sites may require 2–3 on-site days for comprehensive coverage. Brownfield sites with poor documentation may require extended discovery effort. Site survey is often combined with SK-D14-012 (existing infrastructure inventory) in a single site visit to optimize travel.
