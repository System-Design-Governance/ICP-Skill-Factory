## Description

Orchestrate the complete Pre-Gate 0 input preparation workflow, ensuring all 5 mandatory Gate 0 inputs are produced, quality-checked, and delivered within the governance-defined boundary. Under GOV-SD, the Pre-Gate Design Support Role produces 5 specific Gate 0 inputs: (1) requirement clarification record, (2) risk pre-disclosure list, (3) feasibility assessment, (4) CBOM (commercially usable / design non-binding), and (5) concept architecture (conceptual / non-binding). All Pre-Gate 0 outputs are explicitly advisory — discussions before Gate 0 approval do NOT constitute formal requirement acceptance, resource commitment, or delivery obligation. Responsibility handover occurs within 15 working days of Gate 0 approval. This skill is a composition/orchestration skill that coordinates the individual deliverable skills into a coherent Pre-Gate 0 workflow; each individual deliverable is produced by its own skill (SK-D14-001, SK-D14-003, SK-D14-005, SK-D14-013 ⏳). The package assembly falls under SK-D14-015.

## Inputs

- Customer/employer engagement trigger: tender invitation, RFI, direct inquiry
- All Pre-Gate 0 deliverable skill outputs: requirement clarification (SK-D14-001), risk pre-disclosure (SK-D14-003), feasibility (SK-D14-003), CBOM (SK-D14-005), concept architecture (SK-D14-013 ⏳)
- GOV-SD Pre-Gate 0 framework: 5 mandatory inputs, non-binding boundary, 15-day handover timeline
- GOV-SDP Pre-Gate Design Support Role definition: role scope, KPIs, engagement boundary

## Outputs

- Pre-Gate 0 Workflow Tracker: status dashboard tracking all 5 mandatory inputs through their production stages
- Input Quality Pre-Check: per-input quality assessment before Gate 0 package assembly (SK-D14-015)
- Non-Binding Boundary Verification: confirmation that all Pre-Gate 0 outputs carry appropriate advisory disclaimers per GOV-SD
- Pre-Gate 0 Timeline Plan: schedule for producing all 5 inputs, coordinating site visits, vendor research, and stakeholder meetings within tender deadline
- Handover Readiness Assessment: checklist confirming Pre-Gate Design Support role deliverables are ready for 15-day handover to project team upon Gate 0 approval

## Automation Potential

**Level:** Partial

Workflow status tracking can be automated (task board, milestone tracking). Timeline planning for standard Pre-Gate 0 workflows can be template-driven. Quality checklist verification can be partially automated (document presence and format checking). However, workflow orchestration requires judgment (prioritizing activities, adapting to changing stakeholder availability), non-binding boundary verification requires contextual review (checking that language is genuinely advisory), and handover readiness assessment requires qualitative judgment.

## Acceptance Criteria

1. All 5 mandatory Gate 0 inputs produced and tracked through production stages — workflow dashboard shows 100% completion
2. Non-binding boundary verified: every Pre-Gate 0 output carries explicit advisory disclaimer per GOV-SD — no language implying commitment
3. Pre-Gate 0 timeline met: all inputs produced within tender deadline or agreed engagement timeline
4. Input quality pre-checked: each input assessed against GOV-SD quality thresholds before handover to Gate 0 package assembly (SK-D14-015)
5. Handover readiness confirmed: Pre-Gate Design Support role deliverables documented and ready for 15-day transfer to project team
6. Pre-Gate 0 KPIs tracked: workflow completion metrics aligned with GOV-SDP Pre-Gate Design Support Role KPI definitions
