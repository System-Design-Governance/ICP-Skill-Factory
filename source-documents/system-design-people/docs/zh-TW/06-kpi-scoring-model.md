# KPI 評分模型（含例外加分機制）

## KPI Scoring Model with Exception Bonus Mechanism

**文件編號：** GOV-03-002
**文件版本：** 1.0
**生效日期：** 2026-02-06
**文件層級：** 績效評估文件
**依據文件：** GOV-03-001 角色 SMART KPI 定義

---

## 文件目的

本文件定義各角色之 KPI 權重配置與計分規則，以及受控且可稽核之例外加分機制。此為當責維護型評估框架，非激勵設計工具。

---

## 計分原則

### 基本計分規則

| 原則 | 說明 |
|-----|------|
| 責任強度 > 努力 > 產量 | 高當責項目權重高於純執行項目 |
| 直接當責 > 間接影響 | 角色直接負責之 KPI 權重高於協作性 KPI |
| 系統性失敗影響 > 輕微偏離 | 影響治理完整性之 KPI 權重較高 |
| 基本分總和 = 100 分 | 每角色基本分固定為 100 分 |
| 無零權重 KPI | 每個 KPI 皆須有正權重 |

### 計分方法

| 達成狀況 | 得分計算 |
|---------|---------|
| 達到期望目標 | 該 KPI 滿分 |
| 達到最低可接受 | 該 KPI 權重 × 80% |
| 未達最低可接受 | 該 KPI 權重 × (實際值/最低可接受值) × 80%，下限為 0 |
| 百分比類 KPI 未達 100% 目標但達最低門檻 | 依比例計算 |

### 例外加分原則

| 原則 | 說明 |
|-----|------|
| 每角色每週期上限 | +10 分 |
| 不得彌補基本 KPI 缺失 | 基本 KPI 未達最低門檻者，不得以例外加分補足 |
| 需書面證據與核准 | 所有例外加分皆需文件支持與核准 |
| 不自動授予 | 例外加分為裁量性，非權利 |

## 分數詮釋

### 基本分數詮釋

| 分數區間 | 詮釋 |
|---------|------|
| 95-100 分 | 完全達成當責要求 |
| 85-94 分 | 基本達成，有改善空間 |
| 75-84 分 | 部分達成，需檢討 |
| < 75 分 | 未達最低要求，需改善計畫 |
| 加分後 100+ 分 | 超越基本當責，展現治理領導力 |

### 重要說明

- 例外加分不得使未達最低門檻之 KPI 視為達成
- 基本分低於 75 分者，例外加分不予採計
- 加分紀錄獨立呈現，不掩蓋基本分缺失

## 濫用防範與稽核控制

### 通用防範機制

| 濫用風險 | 防範機制 |
|---------|---------|
| 虛構風險攔截紀錄 | 風險攔截須有獨立第三方確認 |
| 將日常工作包裝為治理資產 | 資產須有至少一個外部複用案例 |
| 事後追溯詮釋決策 | 決策紀錄須於決策時即完成，不接受事後補建 |
| 自我核准加分 | 加分核准權限不得為本人 |

### 稽核軌跡要求

每一例外加分須具備：
1. 申請表單（含事件描述、證據清單、加分類別）
2. 證據文件（可獨立驗證）
3. 核准紀錄（含核准者、日期、核定分數）
4. 異議處理紀錄（若有）

## 假設條件

1. **評估週期**：本模型假設以季為評估週期；年度評估時累計四季結果
2. **紀錄系統**：組織具備支援 KPI 量測與例外加分追蹤之系統
3. **核准權限**：核准者具備獨立判斷能力且無利益衝突
4. **例外加分預算**：組織不設例外加分總額限制，僅設個人上限
