# 績效分數使用規則與管理裁量邊界

## Score Usage and Management Discretion Rules

**文件編號：** GOV-05-002
**文件版本：** 1.0
**生效日期：** 2026-02-06
**文件層級：** 運作規則文件
**依據文件：** GOV-04-001 多角色績效計分與公平聚合模型、GOV-04-002 貢獻對獎酬映射

---

## 文件目的

本文件定義績效分數之合法使用範圍與管理層裁量權邊界，確保：
1. 績效分數之使用具備明確邊界
2. 管理層裁量權受到適當約束
3. 個人權益獲得制度性保障
4. 治理完整性不因裁量而受損

本文件為**強制性規則文件**，所有績效分數之使用皆須遵守本文件規定。

## 績效分數使用邊界

### 分數使用原則

| 原則 | 說明 |
|-----|------|
| 用途明確 | 績效分數僅用於本文件列舉之用途 |
| 不得擴張 | 未列舉之用途不得使用績效分數 |
| 不得曲解 | 分數僅反映設計時權限範圍內之績效 |
| 不得累積 | 分數僅適用於當期，不得跨期累積或抵銷 |
| 不得比較 | 不得將不同評估週期之分數直接比較 |

### 合法使用範圍

#### 可使用績效分數之場景

| 用途類別 | 具體用途 | 使用方式 | 限制 |
|---------|---------|---------|------|
| **獎酬分配** | 獎金池分配 | 依 CRM 層級設定個人上限 | 不得超過 CRM 上限 |
| | 績效獎金計算 | 依 Final Score 或 CRM 層級 | 須遵守映射規則 |
| **晉升考量** | 晉升資格判定 | CRM 層級作為必要條件之一 | 非充分條件 |
| | 晉升優先順序 | 高層級優先考量 | 須綜合其他因素 |
| **發展資源** | 培訓資源分配 | 高貢獻者優先 | 不得排除基本培訓 |
| | 專案選擇權 | 高貢獻者優先選擇 | 須考量專案需求 |
| **績效面談** | 面談內容依據 | 分數作為面談基礎 | 須完整說明計算過程 |
| | 改善計畫觸發 | 未達資格者須制定計畫 | 須提供改善支援 |

#### 禁止使用績效分數之場景

| 禁止用途 | 理由 | 替代方式 |
|---------|------|---------|
| 基本薪資調整 | 分數非薪資決定因素 | 依薪資政策 |
| 解僱決策 | 分數非解僱依據 | 依人事政策 |
| 懲處決策 | 分數非懲處依據 | 依懲處程序 |
| 人格評價 | 分數非人格評價 | 不應評價人格 |
| 公開排名 | 破壞團隊氛圍 | 僅個別溝通 |
| 跨部門比較 | 評估標準不同 | 不應跨部門比較 |
| 跨期累積 | 各期獨立評估 | 僅評估當期 |

### 分數揭露規則

| 揭露對象 | 揭露內容 | 揭露方式 |
|---------|---------|---------|
| 當事人 | 完整計算過程與結果 | 書面報告 + 面談 |
| 直屬主管 | 完整計算過程與結果 | 書面報告 |
| 高階管理層 | 彙總統計與個別結果 | 管理報告 |
| 人資單位 | 獎酬分配相關資料 | 依需要提供 |
| 其他人員 | **不得揭露** | - |

## 管理裁量權邊界

### 裁量權定義

**管理裁量權**：管理層在治理規則框架內，對於未明確規範事項之判斷與決定權。

### 裁量權適用範圍

#### 可裁量事項

| 事項類別 | 可裁量內容 | 裁量者 | 限制 |
|---------|-----------|-------|------|
| **獎酬分配** | 實際獎酬金額（於 CRM 上限內） | 高階管理層 | 不得超過上限 |
| | 獎金發放時機 | 高階管理層 | 須於合理期間內 |
| **晉升決策** | 晉升人選（於合格者中） | 高階管理層 | 須符合 CRM 資格 |
| | 晉升時機 | 高階管理層 | 須有職缺 |
| **發展資源** | 具體資源項目 | 直屬主管 | 不得歧視 |
| | 專案指派 | 直屬主管 | 須考量能力匹配 |
| **面談內容** | 回饋重點 | 直屬主管 | 須基於事實 |
| | 發展建議 | 直屬主管 | 須具建設性 |

#### 不可裁量事項

| 事項類別 | 不可裁量內容 | 理由 |
|---------|-------------|------|
| **分數計算** | KPI 數據、計算公式、權重 | 規則式計算，無裁量空間 |
| | 例外加分資格判定 | 依客觀條件判定 |
| | 聚合公式適用 | 依規則適用 |
| **機制適用** | 治理保護機制觸發 | 強制性機制 |
| | RLS/MREM 適用 | 依客觀條件適用 |
| | CRM 層級對應 | 依映射表對應 |
| **上限突破** | 超過 CRM 獎酬上限 | 上限為強制性 |
| | 超過例外加分上限 | 上限為強制性 |
| **程序遵守** | 數據凍結後追認 | 程序為強制性 |
| | 跳過必要審查 | 程序為強制性 |

### 裁量權行使原則

| 原則 | 說明 |
|-----|------|
| 一致性 | 類似情況應有類似處理 |
| 透明性 | 裁量理由應可說明 |
| 可預測性 | 裁量結果應可預期 |
| 合理性 | 裁量應基於合理考量 |
| 記錄性 | 重要裁量應有紀錄 |

### 裁量權濫用防範

| 濫用類型 | 定義 | 防範措施 |
|---------|------|---------|
| 逾越權限 | 對不可裁量事項進行裁量 | 明確劃定邊界 |
| 恣意裁量 | 無合理理由之裁量 | 要求說明理由 |
| 差別待遇 | 無正當理由之差別對待 | 一致性審查 |
| 報復裁量 | 因非績效因素之不利裁量 | 申訴機制 |

## 績效分數使用邊界詳述

### 獎酬分配使用規則

#### CRM 層級與獎酬上限

| CRM 層級 | 最高獎酬倍數 | 管理裁量範圍 |
|---------|-------------|-------------|
| Not Qualified | 0× | 無獎酬，無裁量空間 |
| Qualified | 0.5× | 0 至 0.5× |
| Solid Contributor | 1.0× | 0 至 1.0× |
| High Contributor | 1.3× | 0 至 1.3× |
| Exceptional Contributor | 1.6× | 0 至 1.6× |

#### 獎酬分配禁止事項

| 禁止事項 | 說明 |
|---------|------|
| 超過 CRM 上限 | 不得以任何理由超過 |
| 無視 CRM 層級 | 不得忽略層級差異 |
| 倒置層級 | 低層級不得高於高層級 |
| 事後調整 | 獎酬確定後不得調降 |

#### 獎酬分配裁量指引

| 考量因素 | 說明 | 權重建議 |
|---------|------|---------|
| CRM 層級 | 最重要考量 | 主要依據 |
| 團隊績效 | 團隊整體表現 | 次要考量 |
| 組織獎金池 | 可分配總額 | 限制因素 |
| 特殊貢獻 | 未納入 KPI 之貢獻 | 須經核准 |

### 晉升考量使用規則

#### CRM 層級與晉升資格

| CRM 層級 | 晉升資格 | 說明 |
|---------|---------|------|
| Not Qualified | 不具資格 | 須先改善績效 |
| Qualified | 有條件資格 | 須連續兩週期達此層級 |
| Solid Contributor | 標準資格 | 符合基本條件 |
| High Contributor | 優先資格 | 優先考量 |
| Exceptional Contributor | 最優先資格 | 強烈建議考量 |

#### 晉升決策裁量指引

| 考量因素 | CRM 層級使用方式 |
|---------|-----------------|
| 資格篩選 | CRM 層級作為必要條件 |
| 優先排序 | 高層級優先考量 |
| 最終決策 | 綜合能力、職缺、發展潛力 |

#### 晉升決策禁止事項

| 禁止事項 | 說明 |
|---------|------|
| 忽略 CRM 資格 | 不得晉升不具資格者 |
| 僅依 CRM 決策 | CRM 非唯一考量 |
| 跳過高層級 | 不得無理由跳過高層級者 |

### 發展資源使用規則

#### 資源分配優先順序

| 優先順序 | CRM 層級 | 資源類型 |
|---------|---------|---------|
| 1（最優先） | Exceptional Contributor | 高階培訓、重點專案 |
| 2 | High Contributor | 進階培訓、挑戰專案 |
| 3 | Solid Contributor | 標準培訓、一般專案 |
| 4 | Qualified | 基礎培訓、支援專案 |
| 5 | Not Qualified | 改善計畫、基礎培訓 |

#### 資源分配禁止事項

| 禁止事項 | 說明 |
|---------|------|
| 排除基本培訓 | 不得以低層級排除基本培訓 |
| 歧視性分配 | 不得基於非績效因素分配 |
| 懲罰性分配 | 不得以資源分配作為懲罰 |

### 績效面談使用規則

#### 面談內容要求

| 面談內容 | 說明 |
|---------|------|
| 分數說明 | 須完整說明計算過程 |
| 證據引用 | 須說明證據來源 |
| 層級對應 | 須說明 CRM 層級意涵 |
| 發展建議 | 須提供具體建議 |

#### 面談禁止事項

| 禁止事項 | 說明 |
|---------|------|
| 隱瞞計算過程 | 須完整揭露 |
| 人格攻擊 | 分數非人格評價 |
| 威脅恐嚇 | 不得以分數威脅 |
| 公開比較 | 不得與他人比較 |

## 特殊情境處理

### 績效改善計畫

#### 觸發條件

| 條件 | 處理 |
|-----|------|
| CRM 層級 = Not Qualified | 強制進入績效改善計畫 |
| 連續兩週期 Qualified | 建議進入績效改善計畫 |
| 觸發 Governance Failure Flag | 強制進入績效改善計畫 |

#### 改善計畫內容

| 項目 | 說明 |
|-----|------|
| 問題識別 | 分析績效不足原因 |
| 目標設定 | 設定具體改善目標 |
| 行動計畫 | 制定改善行動 |
| 支援措施 | 提供必要支援 |
| 追蹤機制 | 定期追蹤進度 |
| 期限設定 | 設定改善期限 |

#### 改善計畫裁量指引

| 可裁量 | 不可裁量 |
|-------|---------|
| 改善期限長度（合理範圍） | 是否進入計畫（強制者不可免除） |
| 具體行動內容 | 計畫完整性要求 |
| 支援資源項目 | 基本支援提供 |
| 追蹤頻率 | 追蹤機制存在 |

### 申訴處理

#### 可申訴事項

| 事項 | 申訴依據 |
|-----|---------|
| 數據錯誤 | 發現計算錯誤 |
| 證據遺漏 | 發現應採計之證據遺漏 |
| 程序違規 | 發現程序未被遵守 |
| 裁量濫用 | 發現裁量權濫用 |

#### 不可申訴事項

| 事項 | 理由 |
|-----|------|
| KPI 定義 | 定義為治理決策 |
| 權重設定 | 設定為治理決策 |
| CRM 映射表 | 映射表為治理決策 |
| 評估結果本身 | 若計算正確則不可申訴 |

#### 申訴程序

```
Step 1: 當事人提出書面申訴（含具體事由與證據）
        ↓
Step 2: Governance Lead 初審（5 個工作日內）
        ↓
Step 3: 若屬可申訴事項，進入調查
        ↓
Step 4: 調查結果報告
        ↓
Step 5: Head of System Design 裁決
        ↓
Step 6: 若涉及數據錯誤，更正並記錄
```

### 例外處理

#### 可申請例外之事項

| 事項 | 核准者 | 條件 |
|-----|-------|------|
| 獎酬分配例外 | 高階管理層 | 須有正當理由並記錄 |
| 晉升資格例外 | 高階管理層 | 須有正當理由並記錄 |
| 週期例外 | Head of System Design | 須有不可抗力因素 |

#### 不可申請例外之事項

| 事項 | 理由 |
|-----|------|
| 分數計算規則 | 規則為強制性 |
| 治理保護機制 | 機制為強制性 |
| CRM 層級對應 | 映射為強制性 |
| 上限突破 | 上限為強制性 |

## 附錄 B：常見問題

### Q1: 主管可否因個人好惡調整獎酬？

**A1**: 否。獎酬須依 CRM 層級設定上限，於上限內之裁量須基於合理考量，不得基於個人好惡。

### Q2: 績效分數可否用於解僱決策？

**A2**: 否。績效分數不得作為解僱依據。解僱須依組織人事政策另行處理。

### Q3: 可否公開績效排名？

**A3**: 否。績效分數僅向當事人、直屬主管、高階管理層、人資單位揭露，不得公開排名。

### Q4: 主管可否超過 CRM 上限給予獎酬？

**A4**: 否。CRM 上限為強制性，不得以任何理由超過。

### Q5: 績效面談可否不說明計算過程？

**A5**: 否。績效面談須完整說明計算過程，當事人有權了解分數如何計算。

### Q6: 可否將今年高分抵銷去年低分？

**A6**: 否。每個評估週期獨立評估，不得跨期累積或抵銷。

## 假設條件

1. **政策存在**：假設組織有既定之薪資、晉升、懲處等人事政策
2. **稽核能力**：假設組織具備內部稽核能力
3. **申訴機制**：假設組織具備申訴處理程序
4. **紀錄系統**：假設組織具備紀錄保存系統
